Unlock Claude Code's Thinking Modes: From 'think' to 'ultrathink'

Jul 28, 2025
Vibe Sparking

When used well, think is your secret weapon for breaking through bottlenecks; when used poorly, Claude works like it “left its brain at home”.

Have You Ever Had This Experience?

Section titled “Have You Ever Had This Experience?”

You ask Claude to help analyze a problem, and it gives you an answer in three sentences.

It looks “decent enough”, but when you try to implement it, everything goes wrong, full of bugs.

Many people think Claude isn’t capable, but the truth is: you didn’t let it think properly!

Claude’s Hidden Skill: Thinking Gear Adjuster

Section titled “Claude’s Hidden Skill: Thinking Gear Adjuster”

Claude Code actually has a built-in “thinking accelerator” tool (called Think Tool), but you have to trigger it yourself.

🧩 It supports 4 thinking levels:

Section titled “🧩 It supports 4 thinking levels:”
GearKeywordsThinking Token LimitThinking Depth
NONENo keywords added0 TokensQuick execution, almost no thinking
BASICthink / 想4K TokensBasic thinking, quick rough planning
MIDDLEthink hard / 好好想10K TokensModerate thinking, suitable for complex tasks
HIGHESTultrathink / 仔细思考32K TokensDeepest thinking, suitable for tough challenges

This isn’t mysticism, but real settings written in Claude Code’s engineering code.

Add keywords, and it really will think for a while longer!

🎯Chinese Keywords Can Also Trigger

Section titled “🎯Chinese Keywords Can Also Trigger”

Don’t worry that Claude can’t understand Chinese, its trigger logic also supports our native language:

  • HIGHEST (32K): 仔细思考、深思、多想一会
  • MIDDLE (10K): 好好想、多想想
  • BASIC (4K): 思考、想

For example, if you say “Think carefully about this problem before giving me a plan” — Claude immediately switches to the highest gear, allocating the most thinking time!

Practical Advice: How to Use Most Effectively?

Section titled “Practical Advice: How to Use Most Effectively?”

Of course, here are typical usage scenario examples for each thinking gear, concise and clear, ready to use directly 👇

🧠 ultrathink Example: Breaking Down Complex Product Features

Section titled “🧠 ultrathink Example: Breaking Down Complex Product Features”

Prompt: I want to develop a “Smart Daily Report Assistant” that can automatically aggregate today’s key events from Slack, Notion, GitHub, and help me generate daily report drafts. Please think carefully (ultrathink) about this problem first, then output a module division, data flow, and API interface design.

📌 Use Cases: Multi-system integration, vague requirements, need deep architectural thinking

🧠 think hard Example: Evaluating Multiple Technical Solutions

Section titled “🧠 think hard Example: Evaluating Multiple Technical Solutions”

Prompt: I want to choose a database solution for a multi-user collaborative document application. Please think hard about whether to choose PostgreSQL + Redis or MongoDB + Kafka, output a comparative analysis and give recommendation reasons.

📌 Use Cases: Multiple routes available, need comparative analysis

🧠 think Example: Small Task Logical Planning

Section titled “🧠 think Example: Small Task Logical Planning”

Prompt: I’m planning to write a script that downloads all image links from a Markdown file and renames them. Please think about how to do this first, which Python libraries are most suitable, then write the code for me.

📌 Use Cases: Task isn’t complex, just need a simple run-through of the approach

🧠 No Keywords Example: Direct Action

Section titled “🧠 No Keywords Example: Direct Action”

Prompt: I’ve already written the usage instructions for this AI summary tool. Please optimize the following text format into a README.md file, output the result directly, no need to overthink.

📌 Use Cases: Clear requirements, just need execution, no planning needed

🚀Recommended Usage Pattern: “Explore, Plan, Code, Commit”

Section titled “🚀Recommended Usage Pattern: “Explore, Plan, Code, Commit””

Claude Code officially recommends a 4-step workflow:

  1. Explore: Let Claude read files first, but don’t write code
  2. Plan: Use ultrathink to make detailed plans (core step)
  3. Code: Write code based on the plan, self-check reasonableness
  4. Commit: Generate Pull Request and changelog

📌 Key Point: Step #2 Plan must add keywords to trigger “thinking gears”, otherwise it might skip deep thinking and start working directly!

🧠It’s Not That Claude Isn’t Smart, You Just Didn’t Let It Use Its Brain

Section titled “🧠It’s Not That Claude Isn’t Smart, You Just Didn’t Let It Use Its Brain”

Claude is very capable, but if you don’t give it “thinking space”, it can only respond hastily.

Use think, think hard, think harder, ultrathink well, and you can not only improve output quality but also solve complex problems in one go.
