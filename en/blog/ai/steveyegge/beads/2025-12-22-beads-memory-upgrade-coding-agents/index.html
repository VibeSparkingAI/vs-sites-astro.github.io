Beads: A Memory Upgrade for Your AI Coding Agent

Dec 22, 2025
Vibe Sparking

Introduction

Section titled “Introduction”

Beads (bd) is a distributed, git-backed graph issue tracker designed specifically for AI coding agents. Created by Steve Yegge (former Google/Amazon engineer and renowned software blogger), Beads solves one of the most frustrating problems in AI-assisted development: the context window limitation.

When you start a new session with your AI coding assistant, it loses all understanding of your project goals and progress. Beads provides a persistent, structured memory that replaces messy markdown plans with a dependency-aware graph, allowing agents to handle long-horizon tasks without losing context.

Key Highlights

Section titled “Key Highlights”
  • Git as Database: Issues stored as JSONL in .beads/, versioned and merged like code
  • Agent-Optimized: JSON output, dependency tracking, and auto-ready task detection
  • Zero Conflict: Hash-based IDs prevent merge collisions in multi-agent workflows
  • Invisible Infrastructure: SQLite local cache with background daemon for auto-sync
  • Memory Compaction: Semantic summarization of old tasks to save context window

Prerequisites

Section titled “Prerequisites”

Before installing Beads, ensure you have:

  • Operating System: Linux (glibc 2.32+), macOS, or Windows 11
  • Git: Version control system (for syncing and collaboration)
  • Go 1.24+ (optional, for building from source)

Precautions

Section titled “Precautions”

Alpha Software Warning

Section titled “Alpha Software Warning”

Beads is currently in alpha (v0.9.x). While stable for development:

  • Avoid using in critical production systems until 1.0 release
  • Back up important data regularly
  • Report issues on GitHub

Git Dependency

Section titled “Git Dependency”

Beads relies entirely on Git for synchronization:

  • Ensure git push completes successfully before ending sessions
  • The “plane has NOT landed until git push completes”

Context Window Consideration

Section titled “Context Window Consideration”

While Beads reduces context consumption, complex queries can still use significant tokens. Use targeted queries (bd ready) rather than full dumps.

FAQ

Section titled “FAQ”

Q: How is Beads different from GitHub Issues?

Section titled “Q: How is Beads different from GitHub Issues?”

A: Beads offers:

  • Typed dependencies with semantic meaning
  • Offline-first design with local SQLite
  • Git-native sync without external services
  • Hash-based IDs preventing multi-agent collisions
  • Agent-optimized output (JSON format)

Q: Can I use Beads offline?

Section titled “Q: Can I use Beads offline?”

A: Yes! All queries run against local SQLite. Sync happens via git push/pull when network is available.

Q: What about existing issue trackers?

Section titled “Q: What about existing issue trackers?”

A: Beads complements rather than replaces tools like Jira or GitHub Issues. Use Beads for AI agent memory and local development; sync important issues to your main tracker.

Conclusion

Section titled “Conclusion”

Beads represents a paradigm shift in how we think about AI coding assistants. Instead of treating them as stateless tools that forget everything between sessions, Beads gives them persistent, structured memory that grows with your project.

By leveraging Git’s familiar workflow, providing agent-optimized queries, and implementing intelligent memory compaction, Beads enables truly long-horizon task management. Whether you’re working solo or coordinating multiple AI agents, Beads ensures your coding assistants never lose track of the bigger picture.
