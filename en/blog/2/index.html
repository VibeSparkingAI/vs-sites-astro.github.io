Claude Code 1.0.9x: Permission Doctor, Direct Memory Editing, Vertex Global Endpoints, and More

Sep 3, 2025
Vibe Sparking

A fast series of 1.0.90–1.0.97 updates lands several practical fixes and upgrades: permission rule validation, editable memory files, Vertex global endpoints, proxy compatibility, hot‑reloaded settings, and a task list. The execution loop feels much tighter for engineers.

Introduction

Section titled “Introduction”

Over the past two weeks, Claude Code shipped multiple minor versions. They look small, but each one targets high‑frequency operations: no more fragile permission rules, edit memory files directly, route Vertex via a global endpoint for higher availability, paste images easily on Windows, and see settings take effect immediately. Below is a hands‑on rundown of the most impactful changes, with copy‑ready usage.

What’s New (1.0.90–1.0.97)

Section titled “What’s New (1.0.90–1.0.97)”
  • 1.0.97: /doctor adds permission rule syntax validation with fix suggestions. No more losing time to a stray parenthesis or wildcard.
  • 1.0.94:
    • Vertex supports a global endpoint, improving cross‑region availability.
    • /memory can now open and edit all imported memory files directly, making team guidance easier to organize.
    • SDK adds support for custom tools as callbacks (in‑process MCP / callback hooks), making it easier to plug your own tools into the agent loop.
    • New /todos lists the current task set, helping the plan → act → verify loop feel natural.
  • 1.0.93:
    • Windows adds Alt + V to paste images from clipboard for fast inclusion of error screenshots and sketches.
    • Supports NO_PROXY to bypass specific hosts/subnets (with HTTP(S)_PROXY)), better for enterprise networks or custom proxy setups.
  • 1.0.90: Settings changes apply immediately with no restart. Note: hooks are an exception (details below).

03 Vertex global endpoint: steadier cross‑region routing (1.0.94)

04 Settings hot‑reload, with a hooks caveat (1.0.90)

Section titled “04 Settings hot‑reload, with a hooks caveat (1.0.90)”

From 1.0.90, most settings apply immediately—no session restart needed. This tightens the tweak → retry loop.

Exception: Hooks (pre/post execution) are not replaced immediately. The current session uses a startup snapshot; after editing hooks, open /hooks to review & apply. Don’t miss this detail.

05 Enterprise‑friendly: NO_PROXY & paste images (1.0.93)

Section titled “05 Enterprise‑friendly: NO_PROXY & paste images (1.0.93)”
  • NO_PROXY support: In complex intranet / multi‑hop proxy setups, directly connect to certain hosts or subnets and bypass the global proxy. Used with HTTP_PROXY/HTTPS_PROXY, this fits CI and bastion/egress scenarios well.
  • Windows Alt + V: Paste images directly in the terminal, ideal for embedding error screenshots/tables into context to speed up “locate → fix”.

06 SDK callbacks & custom tools: wire in your pipeline (1.0.94)

Section titled “06 SDK callbacks & custom tools: wire in your pipeline (1.0.94)”

The TypeScript SDK supports an in‑process MCP server and custom tools, so “build, lint, pre‑publish checks, canary rollback,” etc. can become tools Claude calls. Combine with callbacks/hooks to stitch together success/failure handling and audit logs.

Section titled “Why configure it properly?”

Efficient AI Implementation for Business: Stop Building Demos, Start Building Working Systems

Sep 2, 2025
Vibe Sparking

Executives excited, technology teams frustrated, business units reluctant to adopt. This isn’t a technical problem—it’s an organizational problem. Whether AI implementation succeeds depends on whether the “executive-technology-business” triangular link can form a closed loop.

The following framework = practical playbook. You can directly copy and execute it.

Introduction | The Truth About the “Impossible Triangle”

Section titled “Introduction | The Truth About the “Impossible Triangle””

Over the past two years, from bots to agents, enthusiasm has been high, but projects have been slow to deliver. The reason is often not that models aren’t powerful enough, but that organizational collaboration isn’t aligned: executives want compelling narratives and expect ROI within three months, technology teams want multimodality + compliance + scalability, and business units only care about KPIs and conversion rates.

Real-world data also reminds us: enterprise-level ROI typically takes longer—don’t expect instant results. Forrester and frontline CIO retrospectives indicate that 18–24 months is more common; rushing for quick results often leads to “burning money without results.”

IBM-commissioned 2024/25 research also shows that less than half of enterprise AI projects are profitable in their first year, with most considering cost savings within three years as a reasonable expectation.

AI implementation isn’t a technology race—it’s organizational engineering. MVP, workflow restructuring, governance, and KPI tracking are all essential.

1. Organization First: Getting All Three Parties at the Same Table

Section titled “1. Organization First: Getting All Three Parties at the Same Table”

What should executives do?

  • Appoint a “top-level project” leader, with CEO/General Manager personally overseeing governance and cross-departmental coordination.
  • Define “three lists”: investment boundaries, risk red lines, and metrics to monitor within 90 days.

McKinsey’s global research shows that direct senior leadership responsibility for AI governance and tracking clear KPIs correlate significantly with higher actual profit improvements.

What should technology teams do?

  • Don’t showcase “tool exhibitions”—build “closed-loop toolchains”: Data → RAG/Agent → Analytics → Gradual Rollout → A/B Testing.
  • Integrate compliance and explainability into the pipeline, with data lineage/audit available on demand.

What should business units do?

  • Business objectives must be quantifiable and directly influenced by AI: AHT, FCR, conversion rates, processing time, person-days saved.
  • Business team members participate in prompt co-creation. Latest research emphasizes: prompt quality is as important as the model itself.

2. 90-Day Minimum Closed-Loop Strategy (Directly Executable)

Section titled “2. 90-Day Minimum Closed-Loop Strategy (Directly Executable)”

Day 0–7 | Choose Scenario

  • Select only one, prioritizing “text-intensive, standardized processes, high-volume and measurable” workflows: customer service responses, knowledge retrieval, email/contract drafting, ticket classification.
  • Establish baseline: e.g., customer service AHT, first-call resolution rate, NPS.

Day 8–21 | Build Foundation

  • Data cleaning + permission governance + RAG indexing, first organize “what can be queried” cleanly.
  • Set three types of tracking: quality (accuracy/recall), efficiency (duration/concurrency), effectiveness (conversion/savings).

Day 22–45 | Build MVP

  • Squad structure: Business PO + Tech TL + Data Governance + Legal/Compliance + AI Ops.
  • Business and technology co-write system prompts and few-shot examples, iterate twice weekly.
  • Deploy shadow mode: AI provides suggestions, humans execute.

Day 46–60 | A/B Testing

  • Split 20–30% traffic for comparison, monitor three metrics: AHT, first-call resolution rate, satisfaction.
  • Academic randomized experiments in customer service scenarios show: agents using generative AI achieve 14–15% productivity improvement per person, with greater improvement for newcomers. Design your metric system based on this replication.

Day 61–90 | Gradual Scaling

  • Upon reaching thresholds, move to semi-automation: AI answers routine questions first, humans handle complex issues.
  • Establish change and rollback mechanisms, automatically downgrade when metrics fall below thresholds.

OECD’s 2025 assessment emphasizes: to capture productivity dividends, organization, processes, and strategies must adjust for AI, not just “switching models.”

Realistic Expectations

  • Cost recovery depends on project complexity, 6–24 months is reasonable; over-pursuing “quarterly ROI” is likely counterproductive.

3. Six “Quick-Win” Scenario Categories (with KPIs)

Section titled “3. Six “Quick-Win” Scenario Categories (with KPIs)”
  1. Customer Service Support/Knowledge Retrieval
  • KPIs: AHT↓, first-call resolution rate↑, NPS↑;
  • Key elements: High-frequency Q&A knowledge base + RAG, process templates, quality spot-checks;
  • Supporting evidence: significant productivity improvement, greater benefits for newcomers.
  1. Sales Email/Proposal Drafting
  • KPIs: Reach rate/response rate/appointment rate;
  • Key elements: CRM data → personalized insights → generate draft → human refinement.
  1. Contract/Legal Document Drafts
  • KPIs: Drafting time, per-person processing volume, review round-trips;
  • Key elements: Clause library + prompt templates + approval watermarks;
  • Legal teams no longer just act as “brakes,” but are integrated into the framework as “guardrails”.
  1. Financial Reconciliation/Receipt Extraction
  • KPIs: Verification accuracy, processing time, anomaly recall;
  • Key elements: Structured extraction + rule validation + anomaly ticketing.
  1. Supply Chain/Production Planning Q&A
  • KPIs: Plan variance, stockout rate, decision latency;
  • Key elements: Constraint-based Q&A + explainable reasoning → human approval.

Section titled “How Different People Experience Claude Code”

1) Zero‑code beginners: a steep climb

Section titled “1) Zero‑code beginners: a steep climb”
  • Claude Code isn’t a one‑click “build an app” magic wand. It expects clear natural‑language tasking so it can “plan, write code, and ensure it works.”
  • People with no coding literacy struggle to phrase effective instructions or judge correctness, so CC is “hard to drive.”
  • CC primarily runs in the terminal, assuming some dev environment and command‑line experience — a non‑trivial barrier for beginners.

2) Senior architects: cautiously skeptical

Section titled “2) Senior architects: cautiously skeptical”
  • Many view CC’s value as “limited” — hand‑writing the code feels safer and more reliable.
  • CC can help “debug and fix issues,” scanning a codebase for bugs and proposing patches, but results still require human review.
  • The permissioned workflow asks for user approvals step by step (you can enable Yolo mode to auto‑approve), which can feel inefficient.
  • CC remains early and can be unstable.
  • Net: treat it as an assistant, not a crutch.

Section titled “3) Technical PMs / founders / CEOs: newfound leverage”
