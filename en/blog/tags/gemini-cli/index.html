Your AI Builds Sites That All Look the Same. Impeccable Fixes That.

Section titled “Your AI Builds Sites That All Look the Same. Impeccable Fixes That.”

You have seen this website before. Inter font. Purple gradient. Cards nested inside cards. Gray text on a colored background. It was built by AI, but so was the last one you saw, and the one before that.

Vibe coding in 2026 lets us build anything in minutes. The dirty secret: it all looks the same.

The Sameness Problem

Section titled “The Sameness Problem”

Every major LLM trained on the same web data. The same Bootstrap layouts, the same Tailwind templates, the same UI kit screenshots. Without specific guidance, they converge on the statistical average of “website.”

Ask Claude or GPT to “build me a landing page” and you get:

  • Inter or system font (never anything distinctive)
  • Purple-to-blue gradient hero
  • Cards wrapped in cards wrapped in more cards
  • Gray body text on colored backgrounds (poor contrast)
  • Bounce or elastic animations (looks dated since about 2022)

This is not a bug. It is the most statistically probable output given generic input. The AI is doing exactly what you told it to do. The issue is that you did not tell it enough.

The Vocabulary Gap

Section titled “The Vocabulary Gap”

Paul Bakaus, the creator of Impeccable, frames the problem precisely: great design prompts require design vocabulary. Most developers do not have it.

You cannot ask for “more vertical rhythm” if you have never used those words. You cannot request “tinted neutrals instead of pure grays” if you do not know that is an option. You cannot steer the AI toward “OKLCH color space” or “modular type scale” if those concepts are not in your vocabulary.

The bottleneck is not AI capability. It is the words between you and the AI.

What Impeccable Does

Section titled “What Impeccable Does”

Impeccable is not a component library or a design system. It is a vocabulary layer. Built as skills and commands for AI harnesses (Cursor, Claude Code, Gemini CLI, Codex CLI, Kiro), it extends Anthropic’s original frontend-design skill with deeper expertise.

The system has three parts:

1. An enhanced skill with 7 domain-specific references

ReferenceWhat it covers
typographyType systems, font pairing, modular scales, OpenType features
color-and-contrastOKLCH, tinted neutrals, dark mode, accessibility compliance
spatial-designSpacing systems, grids, visual hierarchy
motion-designEasing curves, staggering, reduced-motion handling
interaction-designForms, focus states, loading patterns
responsive-designMobile-first, fluid typography, container queries
ux-writingButton labels, error messages, empty states

When you install Impeccable, the AI gains access to these references. It knows what good typography looks like, what proper color contrast requires, how spacing should work. The skill fires automatically when you work on frontend code.

2. 17 slash commands

These are the vocabulary in action. Instead of trying to describe what you want, you use a command:

CommandWhat it does
/auditTechnical quality checks — a11y, performance, responsive
/critiqueUX design review — hierarchy, clarity, emotional resonance
/normalizeAlign with design system standards
/polishFinal pass before shipping

/optimizePerformance improvements

Most commands accept an argument to focus on a specific area: /audit header, /polish checkout-form.

3. Curated anti-patterns

This is the part that surprised me. Telling an AI what NOT to do turns out to be just as important as telling it what to do.

Impeccable explicitly instructs the AI to avoid:

  • Overused fonts (Arial, Inter, system defaults without purpose)
  • Gray text on colored backgrounds
  • Pure black or gray (always tint with a hue)
  • Wrapping everything in cards, or nesting cards inside cards
  • Bounce/elastic easing (reads as dated)

Anti-patterns work because LLMs have strong statistical biases toward these exact patterns. An explicit “do not” breaks the default trajectory.

Section titled “A Practical Workflow”

  2. Run /audit to catch technical issues (a11y, responsive, performance)

Each command speaks design language to the AI so you do not have to.

How It Compares

Section titled “How It Compares”

Anthropic’s built-in frontend-design skill laid the groundwork. It gives Claude basic awareness of design principles. Impeccable builds on that foundation by:

  • Adding 7 specialized reference files (typography, color, spatial, motion, interaction, responsive, UX writing)
  • Providing 17 actionable commands instead of relying on general awareness
  • Including explicit anti-patterns that break LLM defaults
  • Working across multiple AI harnesses, not just Claude Code

The difference is between knowing design principles exist and having the specific vocabulary to apply them.

Who This Is For

Section titled “Who This Is For”

If you are a developer using AI coding assistants and you care about design quality but do not have formal design training, Impeccable bridges the gap. It does not replace a designer. It gives you enough design vocabulary to steer AI output from “generic template” to “intentional design.”

If you have been shipping AI-generated UIs and they all look suspiciously similar — same fonts, same gradients, same card layouts — this is why, and this is the fix.

Hands‑On with Vibe Kanban: Manage Claude, Gemini & Amp Tasks in One Board

Jul 12, 2025
Vibe Sparking

Coding Agents (like Claude Code, Gemini CLI, Amp, etc.) are becoming increasingly important in engineering development. Teams are finding that the time spent on planning, reviewing, and coordinating these human tasks now exceeds the time spent writing code itself.

Vibe Kanban https://github.com/BloopAI/vibe-kanban is a new tool specifically designed to solve these collaboration workflow problems. Through its Kanban board interface, it allows you to:

  • Quickly switch between coding agents (Claude, Gemini, Amp, etc.);
  • Call multiple agents in parallel or series, automatically executing tasks;
  • Visualize task status, code reviews, and quickly launch Dev Servers;
  • Centrally manage agent MPC configurations to improve overall collaboration efficiency;
  • In the future, coding agents will be able to automatically create task tickets through MCP servers.

Vibe Kanban’s design philosophy is: When most code is written by AI, our human role becomes planning, reviewing, and coordinating. This is not just a trend, but a completely new way of development work.

⚙️ Installation + Launch Guide

Section titled “⚙️ Installation + Launch Guide”

  1. Pre-configure Agents For example, install and log into Claude Code, Gemini CLI, Amp, etc.

  2. Run command to start Vibe Kanban

    Automatically finds local port and opens browser (e.g., http://127.0.0.1:53046), entering the Kanban workspace.

  3. Configure Agents Initially, you need to add Agent MCP in Settings (e.g., Claude, Gemini, Amp tokens/paths), then you can freely switch agents in the UI to execute tasks.

  4. Create Project Create a new project in the Kanban interface, add Tasks, and you can start using it.

📌 Core Features Explained

Section titled “📌 Core Features Explained”

1. Multi-Agent Parallel or Serial Task Execution

Section titled “1. Multi-Agent Parallel or Serial Task Execution”

After switching to the corresponding agent, you can initiate multiple “Attempts” for the same task. Supports parallel execution (different branches/workspaces) and serial execution (waiting for the previous step to complete before proceeding). Execution status and logs are clear at a glance.

2. Quick Review and Dev Server Launch

Section titled “2. Quick Review and Dev Server Launch”

After each agent generates code, Vibe Kanban automatically displays diffs and execution output, providing a one-click Dev Server launch button for real-time preview.

3. Visual Task Flow Management

Section titled “3. Visual Task Flow Management”

Uses the common three-column Kanban layout (To Do, In Progress, Done), providing an intuitive view of task status and agent execution progress. You can create multiple attempts for a task and mark them as success/failure.

4. Centralized Agent MCP Configuration

Section titled “4. Centralized Agent MCP Configuration”

All agent configurations (e.g., Claude, Gemini tokens, run commands, etc.) are centrally managed in the Settings interface, making switching seamless.

5. Automatic Ticket Creation by Agents (MCP Integration)

Section titled “5. Automatic Ticket Creation by Agents (MCP Integration)”

Vibe Kanban has a built-in “Vibe Kanban MCP” server plugin that can automatically generate planner tickets through agents. For example:

“Plan a migration from AWS to Azure and create detailed tickets…” The agent will automatically break down tasks and create multiple Kanban cards, all done in the UI without manual intervention.

💼 A Typical Workflow Example

Section titled “💼 A Typical Workflow Example”
  1. Create a task in Kanban: e.g., “Implement user login page”;
  2. Select an agent (like Claude Code), click “New Attempt”;
  3. Agent automatically generates login page components and backend interfaces;
  4. You review the differences, run Dev Server, and verify the UI;
  5. If issues are found, modify instructions for the agent to retry, or re-select an agent (like Gemini);
  6. Mark the attempt as successful and move the task to Done.

The entire process requires no switching between command line windows, everything is operated within the UI, efficient and intuitive.

  • For individual developers: Greatly improves task startup, review, and debugging efficiency, making development smoother and more structured.

  • For small team collaboration: Agents automatically break down tasks, multiple attempts collaborate to complete, with independent Git worktrees, can connect to external MCP, suitable for team pipeline collaboration.

  • Investing in future capabilities: As agent capabilities continue to improve, Vibe Kanban can seamlessly adapt to different agents, allowing you to experience the latest model capabilities at any time.
