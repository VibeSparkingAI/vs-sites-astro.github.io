Claude Usage Tracker: Native macOS Menu Bar App for Real-Time Claude AI Usage Monitoring

Section titled “Claude Usage Tracker: Native macOS Menu Bar App for Real-Time Claude AI Usage Monitoring”

For Claude AI power users, keeping track of usage limits is crucial—especially when working with the premium Opus model. Claude Usage Tracker is a beautifully crafted, native macOS menu bar application that provides real-time monitoring of your Claude AI usage limits, built entirely with Swift and SwiftUI.

Official Apple code signed • macOS 14.0+ • ~3 MB • MIT License

What is Claude Usage Tracker?

Section titled “What is Claude Usage Tracker?”

Claude Usage Tracker is a lightweight, native macOS menu bar application that provides real-time monitoring of your Claude AI usage limits. Unlike web-based solutions, this app runs natively on your Mac with:

  • Real-Time Monitoring: Track session (5-hour), weekly, and API console usage
  • Official Apple Code Signing: No security warnings—installs like any normal Mac app
  • Automatic Updates: Built-in Sparkle-powered update system
  • Developer Integration: Terminal statusline for Claude Code users
  • Privacy-First: Local storage, no telemetry, no cloud sync

Getting Started

Section titled “Getting Started”

1. Extract Your Session Key

Section titled “1. Extract Your Session Key”

The app needs your Claude AI session key for authentication:

  1. Open claude.ai and log in
  2. Open Developer Tools (Chrome/Edge: F12 or Cmd+Option+I)
  3. Navigate to Application → Cookies → https://claude.ai
  4. Find sessionKey cookie (starts with sk-ant-sid01-)
  5. Copy the entire value

2. First-Time Setup

Section titled “2. First-Time Setup”
  1. Launch Claude Usage Tracker
  2. The Welcome Wizard appears automatically
  3. Paste your session key and click “Validate”
  4. Wait for confirmation (green checkmark)
  5. Click “Done”

The menu bar icon now shows your live usage statistics!

Ghostty Founder's Vibe Coding: A Conversational Journey with Claude

Ghostty Founder’s Vibe Coding: A Conversational Journey with Claude

Section titled “Ghostty Founder’s Vibe Coding: A Conversational Journey with Claude”

When Mitchell Hashimoto, founder of Ghostty (the modern, GPU-accelerated terminal emulator), wanted to improve the macOS update feature, he didn’t just write code — he had a conversation. In a remarkable session he calls “vibe coding,” Mitchell paired with Claude to iteratively improve the codebase through dialogue, exploration, and shared discovery.

This isn’t about AI replacing developers. It’s about AI as a collaborative partner — and the transcript shows something beautiful: the AI catches things humans miss, the human provides direction and context, and together they build something better than either could alone.

What is “Vibe Coding”?

Section titled “What is “Vibe Coding”?”

Vibe coding is a conversational approach to software development where the developer and AI work together through natural dialogue. Rather than treating the AI as a command-execution tool, the developer treats it as a pair programmer — discussing, exploring, and iterating together.
