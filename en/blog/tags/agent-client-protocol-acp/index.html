Zed Adds ACP for Claude Code and Gemini: Many AI Teammates, One Editor

Aug 29, 2025
Vibe Sparking

Zed turns “agents” into a standard interface (ACP) and invites your command‑line agents into the editor. Gemini CLI is first‑class; Claude Code can run in Zed via a community ACP bridge. Multi‑agent coexistence is the new normal.

TL;DR

Section titled “TL;DR”
  • ACP is “LSP for AI coding agents.” Zed open‑sourced the spec (Apache). Any agent that implements ACP can run in Zed.
  • Gemini CLI integration: from Zed 0.201.5, create Gemini threads in the Agent panel; Zed can auto‑install; requires Gemini CLI ≥ 0.2.0.
  • Claude Code works via community bridge acp-claude-code: add one config snippet; supports auto‑accepting file edits permission mode.
  • Feature gaps to note: Gemini CLI currently lacks message history editing/restore, checkpoints, and SSH project usage — on the roadmap.
  • Data stays local: third‑party agent traffic does not tunnel through Zed servers. Multi‑buffer visual diffs and live previews are excellent.

Why This Matters

Section titled “Why This Matters”

  1. Editor–Agent decoupling Like LSP for languages, ACP lets you choose which agent plays. In the same project, switch agents for different tasks anytime.

  2. Real CLI power, visualized in the editor Zed spawns real Gemini CLI subprocesses and speaks JSON‑RPC. You get all CLI capabilities plus live change visualization and multi‑buffer reviews.

  3. Open spec and growing ecosystem ACP is open (Apache). The Neovim community is also adding ACP‑compatible agents. This isn’t single‑editor locked.

Recommended Scenarios

Section titled “Recommended Scenarios”
  • Large‑repo refactors and batch fixes: multi‑buffer review + live diffs = changes at a glance.
  • Multi‑agent “PK”: Gemini drafts refactor plans; Claude Code adds tests and commit messages — all inside one editor.
  • Security/compliance teams: external agent traffic doesn’t traverse Zed servers.

Zed Adds ACP for External Agents (Gemini CLI Live)

Aug 28, 2025
Vibe Sparking

Zed introduced an Agent Client Protocol (ACP) to bring external coding agents like Gemini CLI (and later, Claude Code, etc.) directly into the editor. The first end-to-end integration is Gemini CLI. You keep using the familiar CLI; Zed embeds it in a panel so you can see file changes, review diffs across multiple buffers, and accept or modify patches. ACP is open source (Apache 2.0), welcoming more agents and editors to join the ecosystem.

TL;DR

Section titled “TL;DR”
  • ACP is like LSP for agents: it decouples agents from editors so you can swap them freely.
  • Gemini CLI runs inside Zed now: open the Agent panel to start new threads; supported since Zed 0.201.5+.
  • It’s the real CLI, not a cut‑down version: Zed spawns Gemini CLI and talks via ACP; you can preview files and diffs in the editor UI.
  • Works out of the box: if the CLI isn’t installed, Zed installs it; if it is, Zed picks it up from PATH; Gemini CLI must be ≥ 0.2.0.
  • Bring any agent that speaks ACP: configure and run; debugging logs included.

Introduction

Section titled “Introduction”

Many folks ask: can I switch multiple AI coding agents within a single editor? Zed’s answer: yes—right now. ACP decouples “agent” from “editor.” Gemini CLI leads as the first integration, and Neovim already joined the ACP ecosystem via Code Companion. The ecosystem door is open.

Why ACP Matters

Section titled “Why ACP Matters”

1) Decoupling like LSP You don’t have to change editors to change agents. Any agent that speaks ACP can plug into Zed’s UI, benefiting from multi‑buffer review, syntax highlighting, and visual diffing. Data stays local—nothing is proxied through Zed’s servers.

2) Real capability + editor experience Zed doesn’t simulate a chat box; it runs Gemini CLI as a child process and communicates over JSON‑RPC. You get the CLI’s full capabilities alongside editor‑grade change previews and navigation.

3) Open ecosystem ACP is open source (Apache 2.0). The Zed team is inviting other editors to join; Neovim’s Code Companion already supports ACP‑compatible agents. Expect similar experiences across different editors over time.
