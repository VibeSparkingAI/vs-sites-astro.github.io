HeartPetBond App Launch: GLM 4.7 AI Autonomous Development Passes App Store Review in 2 Days

Jan 12, 2026
Vibe Sparking

First of its kind? This may be the first App developed autonomously by domestic GLM 4.7 LLM to pass App Store review and launch.

Review Passed: 2 Days, First Attempt

Section titled “Review Passed: 2 Days, First Attempt”

Submitted on January 10th, approved on January 12th — taking just over 2 days.

If not submitted on a weekend (January 10th was Saturday), the review time might have been even shorter. Compared to the previous MoneyMind App’s first launch which experienced 14 days, 4 submissions, and 3 rejections, HeartPetBond’s review process was smooth sailing.

Why Did It Pass on First Try?

Section titled “Why Did It Pass on First Try?”

Experience Reuse: Learning from MoneyMind’s Pitfalls

Section titled “Experience Reuse: Learning from MoneyMind’s Pitfalls”

Since I had successfully launched MoneyMind App before, I had encountered many pitfalls. This time, I proactively avoided these issues:

  • Screen Compatibility: MoneyMind was rejected for text overlap issues. This time, AI-autonomous testing prioritized verification across different screen sizes
  • Account Deletion: App Store requires account deletion functionality. This time, it was built-in from the start
  • Feature Completeness: Ensured all promoted features work properly, with no “coming soon” placeholders

Tech Stack: GLM 4.7’s First App Store Debut

Section titled “Tech Stack: GLM 4.7’s First App Store Debut”

HeartPetBond was developed autonomously using domestic Zhipu AI’s GLM 4.7 Large Language Model:

ComponentDescription
Claude CodeBest engineered AI Agent system
GLM 4.7Domestic excellence, extremely cost-effective (200K context, 64K output)
BMAD V6Most powerful AI-driven agile development process
Flutter MCPBest cross-platform development MCP support

Compared to using Opus 4.5 for MoneyMind development, GLM 4.7 costs only about 1/7, but under the BMAD agile process, it’s fully capable of handling App development tasks.

Guideline 5.1.1(v) - Account Deletion

The Core Requirement

Creating a Distribution Certificate

Section titled “Creating a Distribution Certificate”
  1. Open Keychain Access on your Mac
  2. Select Keychain Access > Certificate Assistant > Request a Certificate from a Certificate Authority
  3. Enter your email and select “Saved to disk”
  4. Go to Apple Developer Portal
  5. Navigate to Certificates, Identifiers & Profiles
  6. Click + to create new certificate
  7. Select Apple Distribution
  8. Upload your Certificate Signing Request (CSR)
  9. Download and double-click to install

Common Rejection Reasons & Solutions

Section titled “Common Rejection Reasons & Solutions”

1. Guideline 2.1 - App Completeness

Section titled “1. Guideline 2.1 - App Completeness”

Issue: App crashes, has placeholder content, or incomplete features.

Solution: Thoroughly test before submission. Remove any TODO or placeholder text.

2. Guideline 2.3 - Accurate Metadata

Section titled “2. Guideline 2.3 - Accurate Metadata”

Issue: Screenshots don’t match actual app, misleading descriptions.

Solution: Update screenshots after every major UI change.

3. Guideline 5.1.1 - Data Collection and Storage

Section titled “3. Guideline 5.1.1 - Data Collection and Storage”

Issue: Missing privacy policy or improper data handling disclosure.

Solution: Create comprehensive privacy policy, declare all data collection in App Store Connect.

4. Guideline 4.2 - Minimum Functionality

Section titled “4. Guideline 4.2 - Minimum Functionality”

Issue: App is too simple or doesn’t provide enough value.

Solution: Ensure your app provides meaningful functionality beyond what a website could offer.
