JJYB AI Smart Cut: Intelligent Video Auto-Editing with AI Commentary, Offline TTS, and Voice Cloning

Dec 29, 2025
Vibe Sparking

Introduction: The Complete AI Video Editing Suite

Section titled “Introduction: The Complete AI Video Editing Suite”

Video content creation has exploded, but the editing workflow remains time-consuming and technically demanding. JJYB_AI_VideoAutoCut (JJYB AI Smart Cut) is a comprehensive, locally-deployable AI video editing platform that automates the entire pipeline—from visual analysis and script generation to voice synthesis and final video composition.

This isn’t just another video editor. It’s a complete AI-powered production suite with 143 configuration options, support for 19 different AI models, and 100% offline capability for sensitive content.

What is JJYB_AI_VideoAutoCut?

Section titled “What is JJYB_AI_VideoAutoCut?”

JJYB_AI_VideoAutoCut is an open-source, intelligent video editing system developed by jianjieyiban. It combines computer vision, large language models, and text-to-speech synthesis into a unified workflow for automated video content creation.

GitHub Repository: jianjieyiban/JJYB_AI_VideoAutoCut

License: MIT License

Status: 100% Complete (v2.0)

Platform: Windows 10/11 (64-bit)

Section titled “Three Core Modules”

1. Video Editor

Section titled “1. Video Editor”

A full-featured web-based video editing interface with:

  • Complete playback controls with track management
  • Triple synchronization (audio-video, text-video, text-audio) under 100ms
  • Effects, filters, and transitions support
  • Real-time preview at http://localhost:5000/

2. AI Voiceover (TTS)

Section titled “2. AI Voiceover (TTS)”

Multi-engine text-to-speech synthesis supporting:

  • Edge-TTS (23+ voices, free)
  • Google TTS (100+ languages)
  • Azure TTS (professional, paid)
  • Voice Clone (local voice cloning)

With 38 configuration options for acoustic modeling, vocoder settings, and emotional TTS.

3. Original Commentary Generation

Section titled “3. Original Commentary Generation”

Complete AI-powered automated commentary workflow:

Features 52 configuration options for multimodal feature extraction, timeline optimization, and cross-platform adaptation.

4. Video Remix Mode

Section titled “4. Video Remix Mode”

Two remix modes with 34 configuration options:

  • Mass Remix: Batch import videos, AI-identified highlights, style selection (energetic/healing/funny), automatic transitions
  • Music Beat Remix: Rhythm detection (4 algorithms), beat matching strategies (4 types), intelligent clip selection and sequencing

FAQ

Section titled “FAQ”

Q: Is JJYB_AI_VideoAutoCut free? A: Yes, completely free and open-source under MIT License.

Q: Does it work offline? A: Core video editing works offline. AI features require API connectivity, but pyttsx3 provides offline TTS fallback.

Q: What Python version is required? A: Python 3.9-3.11, with 3.10 being recommended for best compatibility.

Q: Can I use my own voice? A: Yes, the Voice Clone feature allows local voice cloning.

Q: How accurate is the video analysis? A: Depends on the vision model selected. GPT-4V and Claude Vision offer the best understanding.

Q: Can it handle batch processing? A: Yes, the remix mode supports batch video import and processing.

  • Context: While handling inbound emails (parsing, creating records, forwarding, moving IMAP folders), mailflow occasionally fails the entire processing chain.
  • Goals:
    • Identify the primary trigger of the transaction failure;
    • Explain why all subsequent DB operations report the same error;
    • Restore processing with the smallest, lowest‑risk change;
    • Provide structural improvement suggestions to increase robustness.

Section titled “2. Exploration Path: From Symptoms to Evidence”

Leveraging Claude Code’s code search and cross‑context capabilities, we proceeded “from symptoms to root cause” in order:

  • Locking onto logs and trigger point

    • The logs show the exception stems from the SQLAlchemy Session state: after a prior transaction failure, rollback() was not performed, leaving the current Session invalid.

  • Locating key code paths

    • mailflow/app/db.py: connection and session factory definitions.
    • mailflow/app/mail.py: main business flow, record creation, and commit points.
    • mailflow/app/utils.py: logging and WeCom notification entry notify_and_log.

  • Evidence 1: Module‑level global Session

    • At the top of mail.py there is:
    • Once any flush/commit fails, this global Session becomes invalid, and all subsequent DB operations that rely on it will fail until an explicit session.rollback() is issued.

  • Evidence 2: Missing exception handling and rollback

    • Multiple session.commit() calls lack try/except/rollback protection:
      • Creating the mail record (create_mail)
      • Relating attachments to mail (_relation_attachment_and_mail)
      • Updating status when dealing with request results (deal_requests_result)
      • Large sections of the main flow in process_mail

Conclusion: The first failed DB operation puts the global Session into an invalid state. Because rollback() is missing, all later commits and even queries on that path fail with the same error, forming a “cascade failure”.

3. Reasoning Chain: From Hypothesis to Verification

Section titled “3. Reasoning Chain: From Hypothesis to Verification”

  1. Hypothesis: An INSERT/UPDATE exception (possibly from data constraints, transient network issues, or the pool) invalidates the global Session; without rollback(), subsequent steps (status updates, attachment relations, mail moves) all fail.

  2. Code verification:

    • Found the global session = session_factory();
    • Searched for session.commit() call sites — most lack except: session.rollback();
    • This matches the error semantics exactly.

  3. Strategy: Move fast with minimal change to restore stability first, then evaluate structural refactoring.

4. Fix Strategy and Implementation

Section titled “4. Fix Strategy and Implementation”

We prioritized the fix in three levels (P0 → P2):

  • P0 (immediate, smallest change): add session.rollback() in exception handlers so the Session can self‑recover.
  • P1 (comprehensive guard): wrap every commit() with try/except/rollback to avoid any single failure taking down the rest of the flow.
  • P2 (structural): replace the module‑level global Session with scoped_session to isolate session state across processing flows (recommended, not mandatory for the hotfix).

  • Add rollback in the main flow (mailflow/app/mail.py)

5. Validation and Regression

Section titled “5. Validation and Regression”

Suggested validation steps:

Section titled “6. Lessons Learned and Best Practices”

  4. Smallest change first: add rollback() so the Session can recover and stop the bleeding.

  • Threshold alerts: flag outliers (e.g., beyond ±3σ) or sudden jumps

Section titled “4) Cross‑checker: SEC XBRL parity”

Use data.sec.gov XBRL APIs for companyfacts or companyconcept: CIK (Apple: 0000320193) → us-gaap/NetIncomeLoss, us-gaap/Revenues, us-gaap/EarningsPerShareDiluted. Compare basis, units, and periods; mismatches go to human review.

The SEC provides official XBRL JSON endpoints and asks clients to respect ≤10 req/s and custom User‑Agent.

5) Trustworthiness & risk controls

Section titled “5) Trustworthiness & risk controls”
  • Units and abbreviations: unify millions/thousands/per‑share; map XBRL units (USD, USD/shares).
  • Negatives and parentheses: parentheses often indicate negatives; normalize to signed numbers.
  • Period selection: Q1/Q2 vs FY — ensure date ranges match.
  • Caching & trace: store extraction snapshots, source URLs, timestamps, and content hashes.
  • Polite crawling: throttle for SEC, set a UA with company+email; queue retries when needed.
