Smart APK Naming for Flutter Android Builds with Claude Code

Sep 17, 2025
Vibe Sparking

Make APKs readable at a glance: channel, environment, version, branch, timestamp — no manual renaming.

Let Claude Code inject the naming rule into android/app/build.gradle. Every build outputs a descriptive filename.

Example output: TestAApp-release-v1.0+1-main-20250918-102530.apk Includes: app name + build type + version name/code + git branch + timestamp.

— Official Flutter build paths and Gradle references are listed at the end.

Why rename the APK file?

Section titled “Why rename the APK file?”

By default it’s app-release.apk. When sending to QA or uploading to CI artifacts, everything has the same name. It’s guesswork to identify versions — frustrating and error-prone.

With a descriptive name you immediately know:

  • Which branch produced it
  • Which version it is
  • When it was built

Productivity goes up immediately.

What you need

Section titled “What you need”

  • A standard Flutter/Android project.

  • Any Android Gradle Plugin (AGP) version works, but naming approaches differ. The robust approach is to copy/rename after assemble. That’s the path used below and is widely adopted under newer AGP APIs.

  • Flutter’s --dart-define values are passed via Gradle -Pdart-defines as Base64. We can decode and read them in build.gradle.

Copy and use: add this to android/app/build.gradle

Section titled “Copy and use: add this to android/app/build.gradle”

Place under the android { ... } section with applicationVariants (Groovy syntax). Idea: get branch name + decode --dart-define + build the new file name + copy/rename after assemble. This “copy and rename” technique is more reliable across AGP versions.

Notes:

  • Local uses git rev-parse; CI can fallback to the GIT_BRANCH environment variable.
  • Newer AGP often breaks “rename in place”; copy+rename is the portable approach.
  • See Flutter docs for build configuration locations and packaging steps.

How to pass --dart-define

Section titled “How to pass --dart-define”

Command example:

Flutter encodes these --dart-define entries in Base64 and injects them as -Pdart-defines. The Gradle snippet above decodes them into a Map.

Common pitfalls (pre‑solved here)

Section titled “Common pitfalls (pre‑solved here)”

  • In‑place rename sometimes fails New AGP Variant APIs changed. Community practice favors “copy + rename after assemble”.

  • Flutter output directory quirks Some builds land under flutter-apk/ or ignore outputFileName. Use the copy+rename fallback.

  • CI with detached HEAD git rev-parse can’t read the branch. Use env fallback (GIT_BRANCH or CI‑specific vars).

Wrap‑up

Section titled “Wrap‑up”

Bake the naming rule into the build step so every artifact is self‑describing. QA hand‑off, tracebacks, and bug hunts all get easier. Claude Code turns these “small but annoying” chores into a one‑time setup you can reuse.
