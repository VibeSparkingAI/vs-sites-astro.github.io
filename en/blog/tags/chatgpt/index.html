Claude Code Best Practices: The Prototyping Method

Sep 2, 2025
Vibe Sparking

First, use Claude Code to quickly “get a prototype working” to refine the requirements; then, return to engineering design and implementation to ensure quality and stability. Efficiency and quality, we want both.

Introduction

Section titled “Introduction”

The hardest part of product development isn’t writing code. It’s clearly defining “what needs to be built.”

That’s why I recommend this two-stage approach: Build a prototype in the first version, and handle the engineering seriously in the second. Claude Code is well-suited for this strategy: it works alongside you in the terminal, handling the grunt work first, allowing you to refine it later.

The 2 Stages of the Prototyping Method

Section titled “The 2 Stages of the Prototyping Method”

Stage 1: Prototype Only, Aim for “It Works”

Section titled “Stage 1: Prototype Only, Aim for “It Works””
  • Goal: Get the feature running to make the requirements tangible.
  • Ignore: Design, performance, security, code quality.
  • Method: Use throwaway code, letting the AI take the lead to quickly explore feasible paths (similar to “Spike/Throwaway Prototyping”).

Why this approach? Because many ambiguities are only exposed when the code is running. Screenshots, GIFs, and clickable prototypes are far more useful than a ten-page requirements document.

Claude Code’s advantages in this stage:

  • Directly read, modify, and run code within your local project—conversation becomes action.
  • Frequent actions can be turned into Slash Commands for one-click reuse, like “start backend + mock data + run locally.”

A word of caution: If you’re using a personal subscription, Anthropic recently introduced an opt-in mechanism for data training that includes Claude Code sessions. Check your privacy settings before prototyping company projects.

Stage 2: Redesign and Engineer Seriously

Section titled “Stage 2: Redesign and Engineer Seriously”
  • Trigger: Requirements are mostly finalized, and the scope is stable.
  • Deliverables: Database schema, API contracts, state management, module boundaries, testing strategy, etc.
  • Pace: Human-led, AI-assisted. You set the design anchors, and the AI builds according to the plan.

How to use Claude Code in this stage:

  • Let it scaffold the project, add boilerplate, and write tests based on your design.
  • @claude in PRs/Issues to automatically create PRs and fix minor issues (via GitHub Actions integration).
  • Write engineering conventions into a CLAUDE.md file or Slash Commands to ensure the AI follows project rules.

Security/Non-Functional Advice: Treat security, performance, and maintainability as “Technical Enablers” and progressively incorporate them into your Definition of Done (DoD) during the second stage. Conduct Spikes for validation when necessary.

When Can You Skip the Prototype?

Section titled “When Can You Skip the Prototype?”
  • The requirements have already been validated (e.g., redoing an old feature, a clear competitor to model after).
  • The primary risk is in the engineering implementation, not the product definition. In this case, you can proceed directly to Stage 2: Design and Implementation, with Claude Code acting as a scaffolding and execution tool.

Field Guide: My Recommended Uses in Daily Development

Section titled “Field Guide: My Recommended Uses in Daily Development”

Here are practical checklists for prototyping to engineering across four common scenarios.

Example A: Growth Experiment Landing Page (Next.js)

Section titled “Example A: Growth Experiment Landing Page (Next.js)”

Prototype Version (Half a day)

  • Have Claude Code generate a minimal page, tracking events, and a form that submits to /api/echo.
  • Get it running with mock data and simple A/B test parameters.

Engineered Version

  • Design the routing and data models, and agree on event names for tracking.
  • Generate OpenAPI specs or protocol documents; write unit/contract tests; integrate the Claude Code Action in CI for automated checks and fixes.

Example B: Mini-Program / BLE Device Data Collection

Section titled “Example B: Mini-Program / BLE Device Data Collection”

Prototype Version

  • Prioritize getting the scan → connect → read → local cache flow working, ignoring exception handling.
  • Use Slash Commands to quickly reuse build and on-device debugging commands.

Engineered Version

  • Define the state machine (connected/disconnected/reconnecting/timeout).
  • Design the offline synchronization protocol and replay strategy; add boundary tests.
  • Create a /test-ble-reconnect command for fault injection and recovery.

Example C: Internal Approval System (Rails/React)

Section titled “Example C: Internal Approval System (Rails/React)”

Prototype Version

  • Focus only on the core workflow: submit, route, approve/reject.
  • Use an in-memory store or SQLite; don’t implement permissions yet.

Engineered Version

  • Design the RBAC and audit log table structures; write migration scripts.
  • Add rate limiting, audit export, and SLO reporting; let Claude Code automatically fix minor issues and add tests in PRs.

Example D: Data Service (FastAPI + DuckDB)

Section titled “Example D: Data Service (FastAPI + DuckDB)”

Prototype Version

  • Start with three read-only APIs for different scenarios, returning static or semi-dynamic data.
  • Ignore caching and permissions to get the frontend unblocked.

Engineered Version

  • Define the pagination/sorting/filtering protocol and error codes; add authentication.
  • Add a caching layer and rate limiting; create a /bench-api command to run load tests against a baseline.

Claude Code’s “Metronome”

Section titled “Claude Code’s “Metronome””
  • Interactive terminal + command line allows for high-frequency iteration by chatting and modifying files.
  • Slash Commands solidify your “operational playbooks” for team sharing.
  • GitHub Actions bring @claude into PRs/Issues for automatic patching, PR generation, and enforcing project standards.

Together, these three features create a fast track from “idea → prototype → engineered product.”

Common Pitfalls and My Evasions

Section titled “Common Pitfalls and My Evasions”
  • Prototype drags on for too long: If requirements aren’t settled in 1-2 weeks, break it down into smaller stories; conduct a Spike summary if needed.
  • Prototype code accidentally shipped to production: Use branch isolation, directory prefixes like /proto-*, and CI checks to block it.
  • Unbounded second stage: Write an Engineering DoD (including security and performance), list non-functional items as Technical Enablers, and complete them sprint by sprint.
  • Privacy compliance: Prioritize enterprise/API channels for company projects; for personal subscriptions, always check data training opt-in settings.

Conclusion

Section titled “Conclusion”

Prototyping isn’t about being lazy; it’s about accelerating clarity. Engineering isn’t about being slow; it’s about reducing long-term costs.

With Claude Code, we can put these two activities in their proper place: AI leads the prototyping, humans lead the engineering. Take that hesitant requirement you have right now, and just get it running.

What do you think? Does your team get stuck more in the prototyping or engineering phase? Let’s discuss in the comments.

Three-Model Playbook for Faster Software Delivery

Aug 31, 2025
Vibe Sparking

Gemini for discovery → ChatGPT for planning and review → Claude Code to generate and execute code → ChatGPT for post-PR review → Gemini for synthesis and knowledge capture. This “division-of-labor” chain connects think → build → review → codify in one loop.

TL;DR

Section titled “TL;DR”
  • Exploration → Gemini: diverge fast, surface patterns you overlooked.
  • Refinement → ChatGPT: turn insights into executable steps and check for logical gaps.
  • Integration → Claude Code: implement, polish, and ensure narrative/architecture coherence.
  • Example: for a product proposal, ask Gemini for deep needs → have ChatGPT list the functional spec → ask Claude Code to assemble a complete, shippable document or PR.

Summary of strengths: Gemini is fast with strong intuition; ChatGPT excels at reasoning; Claude is excellent at structure and clear expression. This also aligns with recent positioning: Google frames Gemini 2.5 as “thinking models” with fast/efficient Flash and stronger Pro; OpenAI highlights ChatGPT’s reasoning and planning for deep research; Anthropic pushes Claude Code as a hands-on terminal agent that can read repos, change code, and open PRs.

Why this split feels natural

Section titled “Why this split feels natural”

  • Gemini: “intuition radar” Native multimodal and thinking-model design → fast and broad coverage. Great for opening up the problem and pulling on promising threads. With 2.5 you can dial thinking budget to go broad first, then deepen.

  • ChatGPT: structured planning and deep reasoning Strong at logical decomposition, argumentation, and integrating external sources — ideal for technical plans, tradeoff analysis, and review checklists.

  • Claude Code: gets things done Built as a terminal-first coding agent: read the repo, change code, run tasks, open PRs and trigger CI. Short path from plan to PR.

Note: model strengths shift over time. Treat this split as pragmatic defaults, not hard rules. Adjust per project type.

A day-to-day development loop (with a realistic example)

Section titled “A day-to-day development loop (with a realistic example)”

Let’s walk a loop using “add a subscription management module to an existing app”.

1) Discovery (Gemini)

Section titled “1) Discovery (Gemini)”

Goal: surface true user pains behind “subscription management”, and produce journeys/boundary conditions.

Example prompt (to Gemini)

“We’re adding ‘subscription management’. From four angles — user segments (new/active/churn-risk), key motivations, cancel/retention touchpoints, and regulatory/payment compliance — map potential needs. Provide 3 high-leverage hypotheses we likely miss, each with a verifiable event and a quantifiable metric.”

Why Gemini: fast divergence/enumeration and adjustable “thinking budget” → go broad first, then narrow.

2) Planning (ChatGPT)

Section titled “2) Planning (ChatGPT)”

Goal: convert “hypothesis → metric → scenario” into system design and milestones.

Example prompt (to ChatGPT)

“Using the 3 high-leverage needs, output a technical solution sketch:

  1. Backend data model and event tables (keys, indexes, hot/cold tiers);
  2. Frontend key wireframes and the state machine;
  3. Risk/compliance interception points;
  4. A/B design and sample-size estimates; Conclude with milestones (two-week sprints) and a risk list (ranked by probability × impact).”

Why ChatGPT: great at structured planning and turning insights into a plan.

3) Code generation and execution (Claude Code)

Section titled “3) Code generation and execution (Claude Code)”

Goal: write code, change code, run tasks, open a PR.

How to use:

  • Install @anthropic-ai/claude-code, enter your repo, and chat; Claude will index the repo and resolve dependencies.
  • In PRs/Issues, mention @claude to generate or modify code and create a PR.

Example command (conceptual)

The value is end-to-end: read code, modify code, run scripts, and drive the Git flow in one loop.

4) Technical review (ChatGPT)

Section titled “4) Technical review (ChatGPT)”

Goal: probe for issues and fix them. Give ChatGPT the Claude-generated PR and ask for a checklist across complexity, testability, rollback strategy, and failure paths, plus concrete Diff suggestions.

ChatGPT’s recent deep-research/long-chain reasoning is well-suited to serve as a contrarian reviewer.

5) Synthesis and knowledge capture (Gemini)

Section titled “5) Synthesis and knowledge capture (Gemini)”

Goal: capture the iteration as postmortem cards and knowledge-base entries. Have Gemini read the PR, issues, CI logs, and alerts, then output:

  • A one-page postmortem (goals, actions, results, learnings, debt);
  • Three stakeholder summaries (ops/support/legal) with different lenses;
  • A next-iteration discovery list and a draft of automated regression cases.

Gemini shines on multi-role summaries and cross-scenario synthesis with speed and breadth.

Real-world tips

Section titled “Real-world tips”
  1. Document the split in the repo In CONTRIBUTING.md, codify the default loop: discover → plan → implement → review → synthesize. Reduce context-switching.
  2. Wire actions into CI/CD Enable Claude Code GitHub Actions to trigger PR generation/modification from comments.
  3. Mobile or Android projects Use Gemini Code Assist inside the IDE for error dissection and fix suggestions.
  4. Mind safety and compliance Follow Google’s GenAI safety guidance: data minimization, explainability, and robust rollback.

Copyable loop checklist (paste into your team wiki)

Section titled “Copyable loop checklist (paste into your team wiki)”
  • Gemini — discovery: segments, journeys, edge cases, key metrics.
  • ChatGPT — planning: architecture/data/state machine, milestones, experiments.
  • Claude Code — implementation: read repo, change code, run tests, branch, PR.
  • ChatGPT — review: complexity, observability, rollback, failure paths.
  • Gemini — synthesis: one-pager, stakeholder summaries, next discovery + regression.

Closing

Section titled “Closing”

Treat models like people with roles, not a generic toolbox. Send the fast-intuitive one to explore, the deliberative one to plan, the hands-on one to ship. When each step has an “owner”, your delivery rhythm flows.

Would this split work for you? How would you tweak it? Drop thoughts in the comments.

Top International AI Products and Services by Domain

Aug 3, 2025
Vibe Sparking

For deep reasoning, multimodal processing, and complex coding, Gemini 2.5 Pro Deep Think is the strongest contender; For comprehensive research and automated information synthesis into reports, choose ChatGPT Deep Research; For terminal-based AI assistance with writing and debugging large project codebases, Claude Code is your most reliable choice; For IDE-based collaborative bug fixing and feature development like working with a teammate, Cursor better fits development workflows; For image creation, use Midjourney; for short video production, rely on Veo 3; For workflow automation and script process interaction, let n8n run autonomously.

This way, choosing the right tools makes everything twice as effective with half the effort. Each product excels in its specialized domain, and combining them strategically maximizes their collective power.

I. ChatGPT o3 (Top Choice for AI Q&A)

Section titled “I. ChatGPT o3 (Top Choice for AI Q&A)”

This is OpenAI’s latest o3 model, primarily excelling at rapid question answering and information retrieval. When you have questions and want quick, accurate answers, or need to discuss knowledge and ask questions, this is your first choice. Fast response, fluent language, perfect for daily use.

II. ChatGPT GPT‑4.5 (Best for AI Writing)

Section titled “II. ChatGPT GPT‑4.5 (Best for AI Writing)”

For creating articles, writing content, and crafting complex text summaries, GPT‑4.5 possesses superior comprehension and expression capabilities. Rich in emotion, clear in structure, diverse in language—ideal for copywriting, reports, or long-form content.

III. Gemini 2.5 Pro (Long Text / Advanced Reasoning)

Section titled “III. Gemini 2.5 Pro (Long Text / Advanced Reasoning)”

Gemini 2.5 Pro is Google’s flagship model, supporting multimodal input including text, images, audio, video, and PDFs. It excels in complex reasoning, code generation, advanced mathematics, and scientific problems, achieving high scores in mathematical or scientific tests. Think of it as the brain-type champion of the AI world, perfectly suited for deep content creation and analysis.

IV. ChatGPT Deep Research (Deep Research Mode)

Section titled “IV. ChatGPT Deep Research (Deep Research Mode)”

This is OpenAI’s model specifically designed for in-depth research, capable of breaking down problems and conducting 5–30 minutes of multi-step web searches, automatically browsing web pages, extracting information, synthesizing analysis, and generating reports with citations. Ideal for market research, paper abstracts, competitive analysis, and industry trend analysis.

V. Midjourney (AI Image Generation)

Section titled “V. Midjourney (AI Image Generation)”

While not shown in personal versions in the chart, Midjourney is the strongest AI tool for generating artistic images, illustrations, and design visuals. Supporting diverse styles, rich details, and exceptional aesthetics, creators frequently choose it for cover images and illustrations.

VI. Claude Code (AI Programming Tool)

Section titled “VI. Claude Code (AI Programming Tool)”

Anthropic’s Claude Code is a command-line terminal AI programming assistant. It can automatically map your project structure, understand codebases, analyze dependencies, and within seconds generate documentation, add new features, or refactor code. Powered by the robust Claude Opus 4 model, it leads the industry in coding performance.

VII. Cursor (AI Programming IDE)

Section titled “VII. Cursor (AI Programming IDE)”

Cursor is an AI-powered IDE based on VS Code that can completely understand and “remember” entire codebases. Give it prompts, and it can generate features, edit code, and answer code-related questions. Many developers report that using Cursor has increased their coding efficiency by multiple times.

VIII. Veo 3 (AI Video Generation)

Section titled “VIII. Veo 3 (AI Video Generation)”

Google’s latest Veo 3 model excels at generating high-quality short videos, supporting text or image input to generate content with original soundtracks, voiceovers, background music, or dialogue, with precise lip-sync, outputting up to 1080p resolution. Enterprises and creators can use it to quickly generate advertisements, product demonstration videos, or educational clips.

IX. n8n (AI Agent / Automation Workflows)

Section titled “IX. n8n (AI Agent / Automation Workflows)”

n8n provides a workflow automation platform that, combined with AI Agents, can build complex workflows: automatically scraping web pages, connecting to databases, calling APIs, letting AI make decisions and execute tasks while continuously operating within systems. Suitable for enterprise scenarios like customer service bots, data processing workflows, and scheduled reminders.

Why This Division?

Scientific recommendation: Adjusting the weekly new vocabulary quantity to around 20-30 words is more reasonable, with flexibility based on the child’s receptive ability. For example, learning 25 new words per week, approximately 5 per day. This ensures children have sufficient repetition and consolidation opportunities without developing apprehension due to excessive quantity. Quality over quantity—ensuring children truly master each word’s spelling and usage is more important than merely pursuing numbers.

III. Age-Appropriate Spelling Memory Techniques

Section titled “III. Age-Appropriate Spelling Memory Techniques”

  • Visual Association Memory: Utilizing children’s love for drawing, connect words with images for memory. For example, when memorizing “elephant,” have children draw an elephant and mark the letters of “elephant” on the elephant’s body; seeing the image triggers letter spelling associations. For abstract words, use homophonics or visualization methods, connecting word shapes with familiar objects, like viewing words as pictures. This visual memory activates the brain’s image memory capabilities, enhancing memory effectiveness.

  • Syllable Breakdown Method: Break long words into readable syllables or phonetic chunks, memorizing section by section. For example, “remember” can be split into “re-mem-ber,” memorizing each section then connecting them for spelling. For multi-syllable words, create sentences to help remember each syllable’s pronunciation and sequence. For instance, memorizing “mammoth” using the sentence “My Aunt Mary’s Manly Hat” to help remember the mam-moth parts (each word’s initial letter corresponds to the original word’s spelling). This breakdown method makes long words “shorter,” reducing memory difficulty.

  • Fun Mnemonics and Initial Letter Aids: Create catchy sentences or mnemonics using word letters’ initials to form memorable phrases. For example, the 7 letters B-E-C-A-U-S-E in “because” can correspond to the English phrase: “Big Elephants Can Always Understand Small Elephants.” Humorous sentences contain each letter’s beginning while having meaning that’s easy to remember, allowing children to memorize spelling through fun. Parents can also work with children to create small mnemonics in Chinese or English for easily misspelled words—the more visual and interesting, the better.

  • Rhyming Song Memory: Use nursery rhymes or verses to remember word spelling and pronunciation. Incorporate spelling rules into simple songs, or find words with similar ending sounds to recite together. For example, use nursery rhymes to remember the rule “letter Q is always followed by U,” or use alphabet songs for spelling. Rhyming helps children better perceive rhythm and letter patterns. For instance, have children discover rhyming word patterns: mastering “ball” spelling makes it easy to remember “tall, call, fall, hall” and other rhyming words’ spelling. Using familiar melodies to sing word letters is also a memory method children enjoy.

  • Multi-sensory Memory: Encourage children to use auditory, tactile, and motor participation in vocabulary memorization. For example, clapping or stomping feet while spelling word rhythms; using colored clay to shape word letter forms; tracing letters on paper before attempting dictation; closing eyes and using fingers to write letters in air or palm. These multi-sensory participation methods enhance children’s muscle memory and sensory memory of letter sequences, making vocabulary memorization more than just static eye and brain activity. Children enjoy hands-on activities, and learning through play creates stronger retention.

In summary, combining images, sounds, rhythm, body movements, and other methods can greatly improve spelling memory efficiency and enjoyment. Parents might work with children to create their own memory techniques, making vocabulary memorization a creative endeavor.

V. Practice Activities and Games to Relieve Dictation Tension

Section titled “V. Practice Activities and Games to Relieve Dictation Tension”

Dictation tension often stems from fear of mistakes and unfamiliar testing atmospheres. We can design targeted practice and games to help children build confidence through simulation and overcome tension through fun.

By integrating spelling practice into games and interactions, such as having children spell words with letter blocks, we can create relaxed practice environments and reduce dictation pressure. Parental companionship and encouragement help children maintain pleasant moods during practice, improving memory effectiveness and confidence.

  • Role-playing Dictation: Parents and children switch roles, playing “teacher and student” games. Have children be “little teachers” giving parents word dictation tests. When children return to being students with parents as teachers simulating dictation, children feel more familiar and relaxed.

Through these activities, children can gradually adapt to dictation formats through simulation and games, turning “testing” into “playing.” When facing actual school dictation, they will associate with relaxed home practice scenes, maintaining calmer attitudes and performing at their proper level.

Section titled “VI. Weekly Review and Testing Recommendations (Identifying and Filling Gaps)”

To truly master learned words, scientific review and testing mechanisms are crucial. Below are recommendations for review and consolidation in weekly cycles:

  • Regular Mini-Tests: Recommend that parents organize weekly mini-tests at weekend or beginning of next week, including new words from the current week plus repeatedly misspelled old words from previous weeks. Formats can vary: both formal written dictation and oral spelling or multiple choice. During tests, try to have children complete independently, then check answers together with parents. Regular testing serves as milestones, checking phased learning effectiveness and urging children to identify and fill gaps.

  • Error Word Notebook and Consolidation: Record misspelled words from each mini-test in an error word notebook. The error notebook can be a notebook or use sticky notes on a wall “error word wall.” At the beginning of the next week, spend 10 minutes with children specifically reviewing these easily misspelled words: analyze why they were wrong—mispronounced or confused letter sequences—and use previously introduced memory techniques to relearn targeting these issues. For example, if letters were carelessly omitted, create mnemonics for reminders. If certain letters are always written backwards, deliberately practice writing more to strengthen muscle memory. This ensures no omissions: weekly gap-filling prevents repeating the same word mistakes in next tests.

  • Cyclical Review (Spaced Repetition): Word memory requires repeated consolidation. Recommend adopting “spaced repetition” strategies for review frequency. For example: new words learned on Monday, quick review on Tuesday (1-day interval), review again on Friday (4-day interval), review again next Monday (7-day interval). During each review, have children attempt eyes-closed spelling or listen to parents read words and write down to see if still remembered. If forgotten, focus on re-memorizing. Some practices show that through progressive consolidation at 1-day, 3-day, 1-week intervals, words transfer from short-term to long-term memory, becoming difficult to forget. Parents can help children create simple review schedules on calendars, marking which days to review which past words by intervals, making reviews planned.

  • Comprehensive Application Review: Besides mechanical spelling memorization, use reading and sentence creation to review words. On weekends, parents can read short stories or passages containing weekly words with children, consolidating vocabulary in context. If suitable materials aren’t available, have children use weekly learned words to each say or write a sentence. For example, if color words were learned this week, have children describe “My favorite color…” Through comprehensive listening, speaking, reading, writing applications, check whether children truly understand word meanings and usage, not just spelling. Such reviews bring words to life and are more interesting.

  • Rewards and Achievement: After weekly reviews and tests, provide children positive feedback and rewards to strengthen achievement feelings. If children successfully master most words, draw progress bars beside vocabulary lists or paste small stars, visually showing “mastered XX words this week.” If all pass, appropriately reward with watching an English animation episode or weekend outdoor activities. Help children feel they’ve gained results after effort, motivating them for next week’s new vocabulary learning.

Through “learning—review—testing—feedback” weekly closed loops, children can promptly discover and remedy vocabulary they haven’t mastered, gradually building more solid word banks. The gap-identification and filling process also cultivates self-monitoring and active learning habits. When children develop weekly autonomous review habits, future English learning becomes easier and more efficient.

Conclusion: This program emphasizes combining scientific memory methods with children’s interests, progressively improving vocabulary and dictation abilities. From controlling weekly word quantities and daily balanced training, to using visual association and fun mnemonics for word memorization, plus parental companionship and encouragement, and techniques for overcoming dictation tension, the aim is helping children learn meaningfully in relaxed, pleasant atmospheres.
