Codex Releases IDE Plugin for VS Code and Cursor

Aug 30, 2025
Vibe Sparking

Want to embed the “code-writing CodeX GPT-5” into VS Code? The Codex IDE gets it right. Read, modify, and run code in one go. This article will guide you through its highlights, installation steps, and most recommended use cases in 10 minutes.

Why Codex IDE?

Section titled “Why Codex IDE?”

It’s an “agent that can run code on its own.” Not just a chat assistant. Codex can directly read files, modify code, and run commands in your working directory. It also supports offloading large tasks to the cloud to run, and then pulling the changes back to your local machine for one-click acceptance. The official extension is compatible with VS Code / Cursor / Windsurf, with stable performance on macOS and Linux. For Windows, using WSL is recommended.

Side-by-side panel for collaboration. Drag the Codex panel to the right side of your editor. Selecting a piece of code allows you to type less and provide more accurate context, resulting in faster and more relevant responses.

Three “approval modes” for a balance of safety and efficiency. The default Agent mode can automatically read, modify, and run files within the current directory; it will ask for permission if it needs to go outside the directory or access the network. You can also switch to the chat-only Chat mode, or the Agent (Full Access) mode which has unrestricted permissions (use with caution).

Cloud execution, local observation. While chatting locally, you can trigger “Run in the cloud” with a single click. For large projects, let it work on a new branch; for smaller tasks, it’s smoother to have it finish up based on your local changes. The cloud context is maintained when you pull the changes back, so you can test them before merging.

Orchestrable and extensible. The extension is built around the open-source Codex CLI and supports the MCP and AGENTS.md specifications, making it easy for teams to write down project rules, build steps, and acceptance procedures for the agent to read.

3 Steps to Get Installed and Logged In

Section titled “3 Steps to Get Installed and Logged In”

① Install the extension Search for “Codex – OpenAI’s coding agent” in the VS Code Marketplace and click install. The process is the same for Cursor and Windsurf.

② Log in It is recommended to use your ChatGPT account (Plus / Pro / Team / Edu / Enterprise are all included). You can also use an API Key, but that requires additional configuration.

③ (Optional) Install the CLI If you want to run it purely in the terminal or for CI, install the Codex CLI: npm i -g @openai/codex or brew install codex, then run codex to get started.

Windows Tip: For a more stable experience, use it with WSL.

6 Immediately Useful Scenarios

Section titled “6 Immediately Useful Scenarios”

1) Quickly Understand Legacy Projects

Section titled “1) Quickly Understand Legacy Projects”

Let Codex run tests, trace call chains, and summarize module responsibilities in the repo to give you an “out-of-the-box tour.” Ideal for new team members or when taking over an old project.

2) Implement Small Features on the Spot

Section titled “2) Implement Small Features on the Spot”

In Agent mode, you can directly ask it to create new files, modify routes, add unit tests, and run local commands. You just need to review the PR.

3) Refactoring/Migration

Section titled “3) Refactoring/Migration”

Provide the goals and constraints, and let it commit changes in batches. If the task is time-consuming or has a large impact, switch to a cloud task to run it, and review the diff as it works.

4) Performance Profiling & Bug Reproduction

Section titled “4) Performance Profiling & Bug Reproduction”

Provide logs and reproduction steps, and let it write probe scripts or local stress test commands and execute them to generate a comparison report.

5) “Internalize” Security and Standards

Section titled “5) “Internalize” Security and Standards”

Add an AGENTS.md file to the project root: specify build/test commands, code style, commit conventions, and security points. The agent will follow these instructions, reducing “verbal agreements.”

6) Improve Team Collaboration Efficiency

Section titled “6) Improve Team Collaboration Efficiency”

Standardize the workflow of “Requirement → Cloud Run → Local Acceptance → Merge” as a team SOP. With everyone following the same process, the delivery pace becomes more stable.

3 Common Workflows You’ll Use

Section titled “3 Common Workflows You’ll Use”

Workflow A: Quick Local Changes

Section titled “Workflow A: Quick Local Changes”
  1. Switch the panel to Agent.
  2. Select code → state your intent → let it modify and run.
  3. Review the diff, and “undo/redo” if you’re not satisfied.

Workflow B: Run Large Tasks in the Cloud

Section titled “Workflow B: Run Large Tasks in the Cloud”
  1. Specify the goal and constraints (branch, test coverage).
  2. Select the environment → Run in the cloud.
  3. You can continue local development during this time; once completed, pull the changes back to your local machine for testing.

Workflow C: Chat First to Finalize the Plan

Section titled “Workflow C: Chat First to Finalize the Plan”

Switch to Chat to think through the design, interfaces, and boundaries first, then switch back to Agent for implementation.

Team Adoption Cheat Sheet (Highly Recommended)

Section titled “Team Adoption Cheat Sheet (Highly Recommended)”

  • Write standards into AGENTS.md: Installation commands, test scripts, linting rules, commit templates, permission scopes. The agent will follow what you write.

  • Choose the mode based on the scenario:

    • Use Chat for design discussions.
    • Use Agent for local changes.
    • Consider Full Access only for large-scale or network-related operations.

  • Use cloud tasks for “heavy lifting”: Migrating frameworks, batch refactoring, generating large test suites.

  • Keep the VS Code panel docked on the right to reduce window switching.

Frequently Asked Questions

Section titled “Frequently Asked Questions”

  • Can’t see it after installation? It might be collapsed in the sidebar’s extension area; drag it to the right to dock it.

  • Planning to use it on Windows? Use WSL first for a smoother experience.

  • How to update? The extension updates automatically, but you can also check for updates manually on the IDE’s extension page.

TL;DR

Section titled “TL;DR”
  • Local = Quick changes/runs, Agent mode is sufficient.
  • Cloud = Heavy lifting, preserves context, pull back for testing when done.
  • Write rules in AGENTS.md, maximizing the quality of team collaboration.

3 “Out-of-the-Box” Example Prompts

Section titled “3 “Out-of-the-Box” Example Prompts”
  • “Add a new /pricing page in apps/web using Tailwind, add e2e tests, and commit after npm run test passes.”
  • “Optimize the N+1 query in UserService using includes, and generate a benchmark comparison report.”
  • “Upgrade eslint and prettier in the Monorepo to the latest stable versions, fix all CI errors, and then create a PR.”

Anthropic (Claude Code’s parent) Nears $10B Funding: An Analysis

Aug 25, 2025
Vibe Sparking

As reported by Bloomberg on August 22, 2025, leading AI startup Anthropic is reportedly close to securing a massive funding round of up to $10 billion — far above prior expectations and among the largest single financing rounds in the AI sector to date.

Based on a comprehensive review of the “Anthropic nearing a $10 billion raise” reports, this analysis applies multi‑source fact‑checking, data validation, and critical reasoning to assess the story’s truthfulness and reliability. We cross‑checked at least 15 sources and found several discrepancies and points of concern. Below, we examine factual verification, data analysis, logical consistency, incentives, market psychology, and potential risks.

Fact‑Check Results

Section titled “Fact‑Check Results”

Funding Size and Timeline Verification

Section titled “Funding Size and Timeline Verification”

Cross‑referencing Bloomberg, CNBC, TechCrunch, Reuters, and other outlets confirms that Anthropic is indeed negotiating a new funding round, but key details vary. Early reporting (e.g., Bloomberg) cited a $3–5 billion raise at a roughly $170 billion valuation, not the $10 billion some outlets repeated.

A more realistic timeline looks like this:

  • 2025‑07‑29: Bloomberg first reported Anthropic’s talks for a $3–5B round at a ~$170B valuation, led by Iconiq Capital, with a potential second lead. The company was also said to be engaging the Qatar Investment Authority (QIA) and Singapore’s GIC for participation.
  • 2025‑08‑20/21: Several outlets, including Bloomberg, updated that the target could rise to “as much as $10B,” potentially becoming one of the largest AI startup financings ever. Anonymous sources cited strong investor demand pushing the target from $5B to $10B.
  • Investor roster: Iconiq Capital as lead appears consistent across reports, but other names vary. Some mention TPG, Lightspeed, Spark Capital, Menlo Ventures, and continued outreach to QIA and GIC. These discrepancies suggest no single official confirmation yet.

Conclusion: A new Anthropic raise is real, but the size and full investor lineup remain fluid. The “doubling” from $5B to $10B hinges on anonymous sourcing and lacks formal confirmation.

Valuation Trajectory Analysis

Section titled “Valuation Trajectory Analysis”

Anthropic’s recent valuation path raises plausibility questions:

  • 2025‑03: Closed a $3.5B round led by Lightspeed at a post‑money of ~$61.5B.
  • 2025‑07: Rumored valuation for the new round rose to ~$170B — a 176% jump in four months.

Annualized, that implies >500% growth, a pace that typically requires extraordinary fundamentals.

Revenue Data: Validation and Caveats

Section titled “Revenue Data: Validation and Caveats”

Annualized revenue run‑rate (ARR)

Section titled “Annualized revenue run‑rate (ARR)”

Multiple sources indicate explosive ARR growth over the past year:

  • Late 2024: ~$1.0B
  • 2025‑05: ~$3.0B
  • 2025‑07: ~$4–5B

Key caveat: ARR often extrapolates “a hot month × 12,” which can materially overstate actual full‑year revenue during hypergrowth.

Customer concentration risk

Section titled “Customer concentration risk”

Anthropic appears to have material concentration in a few customers:

  • Top two customers contribute roughly ~$1.4B ARR (~28%).
  • Largest single customer is the AI coding assistant Cursor at around ~$0.5B.
  • The Claude Code product contributes nearly ~$0.4B ARR.

Loss of one or more of these anchors would significantly impact revenue.

Logic Gaps and Common‑Sense Checks

Section titled “Logic Gaps and Common‑Sense Checks”

Is the raise size reasonable?

Section titled “Is the raise size reasonable?”
  • Cash burn: Raising $3.5B in March and then targeting $5–10B just five months later implies very heavy spend — potentially hundreds of millions to billions per month.
  • Revenue multiple: Anthropic’s valuation/revenue multiple of ~34× vs OpenAI’s ~25× suggests a premium that’s hard to justify without clear differentiation.

Timing coincidences

Section titled “Timing coincidences”
  • Disclosures surfaced nearly in parallel with OpenAI financing chatter, suggesting signaling.
  • Heavy reliance on “people familiar with the matter,” with limited on‑record confirmation.
  • Media “trial balloons” can manufacture urgency and scarcity to lift headline valuations.

Investor Profiles and Geopolitics

Section titled “Investor Profiles and Geopolitics”

Iconiq Capital

Section titled “Iconiq Capital”
  • Strengths: >$80B AUM with ultra‑high‑net‑worth Silicon Valley clientele (e.g., Zuckerberg, Hoffman).
  • Questions:
    • Client‑driven preferences can influence allocation style.
    • Historically favors later‑stage steadier bets; a ~$1B single check would be unusually large.

Sovereign funds and geopolitics

Section titled “Sovereign funds and geopolitics”
  • QIA and GIC involvement introduces geopolitical and national‑security review complexity.
  • Management has previously voiced concerns about optics and regulatory scrutiny tied to Middle East capital, though capital intensity may force pragmatism.

People, Incentives, and Needs

Section titled “People, Incentives, and Needs”
  • Anthropic: Needs substantial capital to compete with OpenAI; lofty valuations help with talent, partnerships, and a potential IPO arc.
  • Investors: FOMO is acute; sovereigns emphasize strategic positioning.
  • Media: “Record‑breaking round” headlines drive traffic; anonymous sourcing lowers the bar, mixing motives.

Market Psychology and Bubble Signals

Section titled “Market Psychology and Bubble Signals”
  • Valuations decoupling from fundamentals relative to traditional tech.
  • Capital concentration: ~64% of US VC dollars in H1 2025 flowed to AI.
  • Herding behavior: Fear of missing the next ChatGPT inflates risks.

Risks and Red Flags

Section titled “Risks and Red Flags”

Technology and commercialization

Section titled “Technology and commercialization”
  • Moat durability: Performance gaps between Claude and GPT‑5 appear to be narrowing.
  • Business model pressure: API pricing wars and sustained compute costs weigh on margins.

Policy and regulation

Section titled “Policy and regulation”
  • US export controls and the EU AI Act may constrain expansion.
  • Sovereign participation could trigger CFIUS review, adding uncertainty.

Conclusion and Recommendations

Section titled “Conclusion and Recommendations”

Fact‑check conclusions

Section titled “Fact‑check conclusions”
  • Directionally true: Anthropic is raising capital.
  • Headline number uncertain: $10B is unconfirmed; $5B looks more likely.
  • Valuation froth: Fundamentals may not support the premium.
  • Incomplete information: Signs of selective disclosure.

Investor takeaways

Section titled “Investor takeaways”
  1. Stay rational about big round headlines.
  2. Prioritize actual revenue and profitability over valuation theater.
  3. Beware bubble dynamics and momentum herding.
  4. Wait for official confirmation before anchoring decisions.

In today’s AI investment cycle, media narratives, corporate signaling, and on‑the‑ground reality often diverge. In moments of mania, noise overwhelms signal. Only disciplined verification and clear‑eyed analysis can keep decisions grounded in truth instead of sentiment.

Introduction

Section titled “Introduction”

Many developers already use Codex to write code, run tests, and edit files. But switching models or reasoning intensity used to mean changing config or restarting. Not anymore. 0.23.0 adds /model with live switching, plus High reasoning for GPT‑5 and even a new Minimal tier. It’s a tangible boost in speed and control.

What exactly changed in this release?

Section titled “What exactly changed in this release?”
  • Slash commands: Type /model in the terminal to change model/reasoning on the fly; /approvals gives you interactive approval control.
  • Reasoning control: Adjust reasoning effort at runtime; GPT‑5 also supports a minimal tier (faster, fewer reasoning tokens).
  • Login and versioning: ChatGPT Plus/Pro/Team accounts can use GPT‑5 directly in Codex; stable release is 0.23.0 (Aug 20).

Background: GPT‑5 exposes a reasoning_effort parameter (minimal | low | medium | high) to control how deeply the model “thinks.” Higher levels usually produce more reasoning tokens with higher latency; minimal trades depth for faster first tokens.

When to use High reasoning vs Minimal/Medium

Section titled “When to use High reasoning vs Minimal/Medium”

Choose High for

  • Cross‑language migrations: e.g., Python → Go while preserving concurrency/error semantics. Demands careful API and model mapping.
  • Tricky bug forensics: deep stacks, race conditions, edge cases—benefit from longer chains of thought.

Choose Minimal/Medium for

  • Scaffolding, bulk lint fixes, renames: you want speed more than deep reasoning. minimal greatly reduces reasoning tokens and speeds things up.

FAQs and small gotchas

Section titled “FAQs and small gotchas”

  • “With ChatGPT sign‑in, is reasoning stuck at Medium?”
    Early community reports said so; but 0.23.0 supports switching /model and reasoning effort at runtime. You can change it directly in the TUI.

    Per the docs, it produces almost no log‑style reasoning tokens and targets scenarios that are extremely sensitive to first‑token latency.

Claude Code Companion: One-Click Pool Switching, Third-Party Model Integration, and Observable Logging

Aug 22, 2025
Vibe Sparking

For those who constantly work with Claude Code, here’s a more stable and transparent “control center.”

Introduction

Section titled “Introduction”

Coding with Claude Code is great, but unstable upstream endpoints, cumbersome switching, and inaccessible logs can be frustrating. Claude Code Companion packages all this into a local API proxy: drag-and-drop sorting for priority-based pool switching, automatic failover on exceptions, plus a visual management console and complete request/response logs. Even beginners can get up and running in 10 minutes.

Why It’s Worth Installing

Section titled “Why It’s Worth Installing”

1) Unstable? Auto-backup Kicks In

Section titled “1) Unstable? Auto-backup Kicks In”

Supports multi-endpoint priority retry + failover. When one pool fails, it automatically switches to the next, and restores the primary when recovered. You’ll barely notice the disruption.

2) Opaque? Full-chain Observability

Section titled “2) Opaque? Full-chain Observability”

Built-in /admin management console lets you add/edit/delete endpoints, view real-time logs, and configure routing rules. Troubleshooting changes from “guessing” to “seeing.”

ChatGPT Agent Review: Smart Reader, But Not Yet a Thinker

Jul 26, 2025
Vibe Sparking

ChatGPT Agent first impression: feels like a diligent but not-so-smart intern.

Have You Ever Encountered This Frustration?

Section titled “Have You Ever Encountered This Frustration?”

Sometimes you come across a seemingly valuable research report or in-depth analysis with an enticing headline, but the content is complex with too much jargon, taking half an hour to read through.

I recently encountered an article titled “How Long Can the Dollar Keep Falling?” from Wall Street Insights. The headline was explosive and the content was quite deep, but I really didn’t have time to read it word by word.

So I had an idea:

Could I use an AI Agent to quickly analyze and summarize it for me?

This led to this experiment—I asked ChatGPT Agent to help me read the article.

What Exactly is ChatGPT Agent?

Section titled “What Exactly is ChatGPT Agent?”

Simply put, this is the “multi-step execution” version of ChatGPT.

Instead of just throwing a prompt and getting back one response, it can make multiple tool calls and complete complex tasks in multiple steps, such as reading an article, calling search functions, extracting viewpoints, doing analysis, etc.

It sounds impressive, and I thought it could help me “delegate” those information-heavy article analysis tasks.

The result?

14 Minutes Later: Usable But Not Practical

Section titled “14 Minutes Later: Usable But Not Practical”

I sent the content of “How Long Can the Dollar Keep Falling?” to the Agent and asked:

Help me extract the article’s viewpoints, provide key logic, and include investment-related reference suggestions.

It did work very diligently,

After waiting 14 minutes, it returned a structured analysis,

Roughly divided into:

  • Short-term and long-term logic for USD weakness
  • Economic data and viewpoints cited by the author
  • Market disagreements on future USD trends
  • Linkage mechanisms between exchange rates and monetary policy

At first glance, the content seemed comprehensive, with summary statements that looked quite professional.

But the biggest problem was: too generic, too textbook-like, lacking “practical operational value”.

For example, I wanted to know:

  • Has it already hit bottom? Are there any signals worth watching?
  • What implications does this have for people holding US bonds, QDII, or US stocks?

The Agent couldn’t answer these questions. It was just restating the article content, without “jumping out” to integrate broader context.

Like a Diligent But Not-So-Smart Intern

Section titled “Like a Diligent But Not-So-Smart Intern”

The whole process made me feel like it was a diligent intern who read through the material and took notes:

  • Can summarize, but doesn’t dare to judge;
  • Can restate, but can’t extract insights;
  • Will say “the article mentions,” but won’t say “this means you might consider xxx.”

Not to mention combining it with my own investment portfolio or current exchange rate data to provide practical advice.

To summarize: the content is reference-worthy, but not very practical.

Can It Still Be Useful? When Is It Most Suitable?

Section titled “Can It Still Be Useful? When Is It Most Suitable?”

Although I was a bit disappointed this time, I think ChatGPT Agent is still useful.

It’s suitable for these scenarios:

  • Quick preview when too lazy to read the full text: Like 10 minutes before a meeting, quickly catching the key points
  • Help organize structure: It’s faster than me at listing paragraph logic and key points, suitable as a foundation for presentation scripts
  • Information overload relief tool: Having it help me read 5 research reports and extract common points—this kind of manual labor it excels at

But expecting it to help me make judgments, make decisions, or formulate investment strategies—it’s still far from that.

Perhaps we need to continue deeper interaction with ChatGPT Agent, providing more contextual information to make it more practical.

AI Isn’t a Crystal Ball, It’s More Like an Accelerator

Section titled “AI Isn’t a Crystal Ball, It’s More Like an Accelerator”

The biggest takeaway from this trial:

AI isn’t someone who thinks for you; it’s an accelerator that helps you quickly “take over repetitive tasks”.

Judgment, understanding, connecting to reality, combining experience—these are still things humans need to do.

In the future, I’ll continue trying to use ChatGPT Agent for information screening, knowledge archiving, document generation and other tasks, rather than expecting it to “think for me” right from the start.
