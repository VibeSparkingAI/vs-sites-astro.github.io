SurfSense: Open Source Alternative to NotebookLM, Perplexity, and Glean

Dec 25, 2025
Vibe Sparking

SurfSense is an open-source AI research agent that combines personal knowledge management with powerful external integrations. Think NotebookLM + Perplexity + Glean—but fully under your control. Self-host it, connect 20+ data sources, and get cited answers without sacrificing privacy. Here’s everything you need to know in 5 minutes.

The modern researcher’s dilemma: NotebookLM is great for documents, Perplexity excels at web search, and Glean conquers enterprise search—but what if you want all three, connected to YOUR data, running on YOUR infrastructure?

Enter SurfSense. In just weeks after launch, it’s become one of the hottest open-source AI projects, offering a unified research experience that respects your privacy.

ConceptWhat It Means
Personal Knowledge BaseUpload 50+ file types (docs, images, videos) and search them semantically
Hybrid SearchCombines semantic + full-text search with Reciprocal Rank Fusion for better results
Cited AnswersEvery AI response includes source citations, just like Perplexity
MCP IntegrationExtensible plugin architecture for custom data sources
Role-Based AccessTeam collaboration with Owner, Admin, Editor, Viewer permissions

Core Highlights

Section titled “Core Highlights”

1. True Data Ownership Unlike cloud-only solutions, SurfSense runs on your infrastructure. Your documents, conversations, and research never leave your control.

2. 20+ Integrations Out of the Box Connect to:

  • Search Engines: SearxNG, Tavily, LinkUp
  • Productivity: Slack, Notion, Gmail, Google Calendar, Airtable
  • Development: GitHub, Linear, Jira, ClickUp, Confluence
  • Media: YouTube, Discord
  • Enterprise: Elasticsearch, BookStack, Luma

3. 100+ LLM Support Works with any model via LiteLLM—OpenAI, Anthropic, Gemini, or local models through Ollama. Switch providers without changing your workflow.

4. Podcast Generation Turn any conversation or research into a podcast. Generates 3-minute audio summaries in ~20 seconds using Kokoro (local TTS), OpenAI, Azure, or Google Vertex AI.

5. Advanced RAG Architecture

  • Hierarchical indices for multi-level document organization
  • Hybrid search combining semantic and keyword matching
  • Multiple reranker options for precision
  • Extensible agent tools registry

Three-Minute Installation

Section titled “Three-Minute Installation”

Option 1: Quick Docker (Fastest)

Access at: http://localhost:3000 (Frontend) and http://localhost:8000 (API)

Option 2: Docker Compose (Production) Full deployment with PostgreSQL, Redis, Celery workers, and pgAdmin for database management.

Option 3: SurfSense Cloud No setup required—sign up at their cloud offering for instant access.

Section titled “Quick Start Workflow”

  1. Launch & Configure After installation, navigate to localhost:3000 and configure your LLM provider (OpenAI API key, or connect Ollama for local inference).

What is Context Engineering?

Section titled “What is Context Engineering?”

Context engineering is the discipline of managing the language model’s context window. The fundamental challenge is that context windows are constrained not by raw token capacity but by attention mechanics.

The Core Problem

Section titled “The Core Problem”

As context length increases, models exhibit predictable degradation patterns:

  • Lost-in-the-Middle Phenomenon - Information in the middle of long contexts gets less attention
  • U-Shaped Attention Curves - Models focus more on beginning and end of context
  • Attention Scarcity - Limited “attention budget” must be allocated wisely

The Solution

Section titled “The Solution”

Effective context engineering means finding the smallest possible set of high-signal tokens that maximize the likelihood of desired outcomes. This requires understanding what context is, how it degrades, and how to optimize it.

Section titled “2. Context Degradation Patterns”

Lost-in-Middle: Critical information placed in the middle of long contexts receives less model attention.

Orchestrator Pattern: A central agent coordinates specialized sub-agents, managing task delegation and result aggregation.

Hierarchical Pattern: Layered agent structures where higher-level agents manage groups of lower-level agents.

Submitting an iOS app to the App Store can feel overwhelming for first-time developers. This guide walks you through the entire process, using our MoneyMind app as a real-world example. Whether you’re building with Flutter, SwiftUI, or UIKit, these steps apply to all iOS app submissions.

  1. App Preparation - Icon, assets, and build configuration
  2. Certificates & Provisioning - Apple Distribution certificate setup
  3. App Store Connect Setup - App information, pricing, and screenshots
  4. Build Upload - Archive and upload via Xcode
  5. Review Submission - Final checklist and submission

Let’s dive into each step.

Icon Specifications

  • Size: 1024 × 1024 pixels

  • Safe Zone: Keep critical elements within 600-700px centered area

Creating a Distribution Certificate

Section titled “Creating a Distribution Certificate”
  1. Open Keychain Access on your Mac
  2. Select Keychain Access > Certificate Assistant > Request a Certificate from a Certificate Authority
  3. Enter your email and select “Saved to disk”
  4. Go to Apple Developer Portal
  5. Navigate to Certificates, Identifiers & Profiles
  6. Click + to create new certificate
  7. Select Apple Distribution
  8. Upload your Certificate Signing Request (CSR)
  9. Download and double-click to install

  5. Select your team

Issue: App crashes, has placeholder content, or incomplete features.

Section titled “2. Guideline 2.3 - Accurate Metadata”

Section titled “4. Guideline 4.2 - Minimum Functionality”
