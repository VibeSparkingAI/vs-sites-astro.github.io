Clawdis: Your Personal AI Assistant on WhatsApp, Telegram & Web

Section titled “Clawdis: Your Personal AI Assistant on WhatsApp, Telegram & Web”

🦞 EXFOLIATE! EXFOLIATE! — That’s the playful battlecry of Clawdis, an open-source personal AI assistant created by Peter Steinberger (@steipete). Unlike cloud-hosted AI services, Clawdis runs on your own devices, connecting to the messaging platforms you already use daily: WhatsApp, Telegram, and Web.

What is Clawdis?

Section titled “What is Clawdis?”

Clawdis is a local-first control plane for your personal AI assistant. Think of it as the infrastructure that lets you interact with an AI agent through:

  • WhatsApp — Send a message, get an intelligent response
  • Telegram — Bot-based conversations via the Telegram Bot API
  • WebChat — Browser-based interface for desktop use
  • macOS app — Menu bar control with voice wake and push-to-talk
  • iOS node — Canvas surface with voice trigger forwarding

The name “Clawdis” is a clever portmanteau: CLAW + TARDIS — because every space lobster needs a time-and-space machine!

Key Features

Section titled “Key Features”

1. Multi-Surface Inbox

Section titled “1. Multi-Surface Inbox”

Clawdis brings your AI assistant to wherever you already communicate. No need to switch to a dedicated app — just message from WhatsApp or Telegram, and your assistant responds.

2. Voice Wake & Push-to-Talk

Section titled “2. Voice Wake & Push-to-Talk”

On macOS and iOS, Clawdis supports:

  • Voice wake — Say the wake word and start talking
  • Push-to-talk overlay — Quick voice interactions
  • Local speech recognition — Privacy-preserving, on-device processing

3. Live Canvas

Section titled “3. Live Canvas”

A visual workspace that you can control from the agent. The Canvas renders live content and supports A2UI (Agent-to-UI) communication for rich interactive experiences.

4. Automation-Ready

Section titled “4. Automation-Ready”

Clawdis comes with built-in capabilities for:

  • Browser control — Automate web interactions
  • Media handling — Process images, audio, and video
  • Tool streaming — Real-time tool execution feedback
  • Smart home integration — Philips Hue, Eight Sleep, and more

Security Considerations

Section titled “Security Considerations”
  • Allowlisting: Restrict who can interact with your assistant via routing.allowFrom
  • Local-first: Gateway runs on 127.0.0.1 by default
  • Credentials storage: WhatsApp session stored in ~/.clawdis/credentials
  • Bridge security: Optional TCP bridge with pairing for mobile nodes

FAQ

Section titled “FAQ”

Q: What AI model does Clawdis use?

Section titled “Q: What AI model does Clawdis use?”

Clawdis is agent-agnostic but was designed around Pi (an RPC-based agent runtime) and commonly uses Claude models. You can configure your preferred AI provider.

Q: Can I use Clawdis for team/business use?

Section titled “Q: Can I use Clawdis for team/business use?”

Clawdis is designed as a single-user, personal assistant. For multi-user scenarios, you’d need to run separate instances.

Q: Is WhatsApp integration against their ToS?

Section titled “Q: Is WhatsApp integration against their ToS?”

Clawdis uses the WhatsApp Web protocol via Baileys. Use responsibly and at your own risk.

Q: How do I add custom skills?

Section titled “Q: How do I add custom skills?”

Create a new folder in ~/clawd/skills/<skill-name>/ with a SKILL.md file describing the skill’s capabilities and instructions.

1. Provide Examples, Don’t Just State Requirements

Section titled “1. Provide Examples, Don’t Just State Requirements”

2. Keep It Simple

Section titled “2. Keep It Simple”

Use action verbs. Avoid obscure expressions. One clear sentence beats three vague paragraphs.

LLMs don’t need pleasantries or context padding. They need direct instructions.

Preferred verbs:

  • Analyze
  • Create
  • Generate
  • Identify
  • Summarize
  • Compare
  • Extract

Before: “I was hoping you could perhaps help me write something that might be useful for understanding the topic of machine learning, if that’s not too much trouble.”

3. Specify Output Format Explicitly

Section titled “3. Specify Output Format Explicitly”

Vague instructions produce random results. If you want JSON, say JSON. If you want 500 words, say 500 words.

The more specific your format requirements, the more predictable your outputs.

Weak: “Generate a blog post about video game consoles.”

Strong: “Generate a 3-paragraph blog post about the history of video game consoles. Use conversational style. Each paragraph should be 4-6 sentences. Include one specific statistic per paragraph.”

4. Use Instructions, Not Constraints

Section titled “4. Use Instructions, Not Constraints”

Tell the model what TO DO, not what NOT to do.

Negation is inherently ambiguous for language models. “Don’t be verbose” is less effective than “Limit response to 100 words.”

Avoid ThisDo This Instead
”Don’t list video game names""Only discuss the console, company, year, and sales figures"
"Don’t be wordy""Limit each paragraph to 3 sentences"
"Don’t make things up""If uncertain, respond with ‘I cannot verify this’”

The psychology: Models process positive instructions more reliably because they define a clear action space rather than an infinite exclusion zone.

5. Control Token Length

Section titled “5. Control Token Length”

Cost scales linearly with tokens. Generating 1000 words costs 10x more than generating 100 words.

Always set explicit length constraints when token efficiency matters.

Techniques:

  • “Limit output to 80 characters”
  • “Explain in a tweet-length message”
  • “Respond in exactly 3 bullet points”
  • “Maximum 50 tokens”

Bonus: Token limits also force conciseness, often improving output quality.

6. Use Variables for Reusable Templates

Section titled “6. Use Variables for Reusable Templates”

Extract changeable elements into variables. One well-crafted template can be reused hundreds of times.

Benefits:

  • Consistency across multiple queries
  • Easier A/B testing
  • Programmatic prompt generation
  • Version control friendly

7. Experiment With Different Input Formats

Section titled “7. Experiment With Different Input Formats”

Questions, statements, and instructions produce different results from identical content.

Test multiple approaches:

  • Question: “What are the benefits of exercise?”
  • Statement: “Exercise has multiple benefits. Explain them.”
  • Instruction: “List 5 benefits of regular exercise with scientific backing.”
  • Role-based: “As a fitness coach, explain why clients should exercise.”

Finding: Lists, paragraphs, and dialogue formats also yield varying stability. Find what works best for your use case.

Section titled “Bonus: Chain of Thought (CoT) Best Practices”

For complex reasoning tasks, set temperature to 0 for rigorous, single-answer problems requiring strict logical reasoning.

Self-Consistency Prompting: Send the same Chain-of-Thought prompt multiple times at high temperature, then “vote” on the most common final answer. This makes answers more robust and reduces hallucinations.

AI coding assistants have become indispensable tools for developers, but their capabilities are often limited to what comes built-in. SkillsMP changes this by providing an open marketplace where developers can discover, install, and share skills across multiple AI platforms.

What is SkillsMP?

Section titled “What is SkillsMP?”

SkillsMP is an open-source marketplace hosting over 31,000 agent skills that work seamlessly with the most popular AI coding assistants:

  • Claude Code - Anthropic’s official CLI tool
  • Codex CLI - OpenAI’s command-line interface
  • ChatGPT - The world’s most popular AI assistant

All skills use the unified SKILL.md format, ensuring compatibility across platforms and making it easy to share capabilities between different AI tools.

The SKILL.md Standard

Section titled “The SKILL.md Standard”

The SKILL.md format is the foundation of the agent skills ecosystem. It defines modular capabilities through:

  • Instructions - Clear guidance for the AI on what the skill does and how to use it
  • Scripts (optional) - Supporting code that extends functionality
  • Templates (optional) - Pre-built structures for common outputs

Key Characteristics

Section titled “Key Characteristics”

Skills are model-invoked, not user-invoked. This means the AI assistant decides when to activate a skill based on the context of your request, rather than requiring explicit commands.
