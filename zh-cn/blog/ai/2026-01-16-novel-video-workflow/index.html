novel-video-workflow：本地运行小说转视频AI工作流，每个人都能成为导演

Section titled “novel-video-workflow：本地运行小说转视频AI工作流，每个人都能成为导演”

简介

Section titled “简介”

在短视频时代，将小说内容转化为视频已成为内容创作的重要方向。novel-video-workflow 由开发者 hulutech-web 创建，提供了一个完全本地运行的AI驱动工作流，能够自动将小说文本转换为带有配音、字幕和场景图像的视频内容，并生成可一键导入剪映的项目文件。

核心价值：完全本地运行、隐私安全、无需云服务费用、支持声音克隆、端到端自动化。

核心功能

Section titled “核心功能”

1. 智能章节分割

Section titled “1. 智能章节分割”

系统自动识别小说文本中的章节标记：

  • 支持中文数字识别（如”第七章”）
  • 支持阿拉伯数字识别（如”第7章”）
  • 自动生成两位数格式输出（chapter_01, chapter_07）
  • 最多支持99章处理
  • 自动检测并跳过重复内容

2. AI驱动文本转语音（TTS）

Section titled “2. AI驱动文本转语音（TTS）”

集成 IndexTTS2 服务实现高质量语音合成：

  • 声音克隆：使用参考音频克隆任何声音
  • 多语言支持：中文语音合成效果出色
  • 高质量输出：生成 WAV 格式音频文件
  • 时间轴同步：为字幕生成提供精确时间戳

3. 自动生成字幕

Section titled “3. 自动生成字幕”

基于音频内容自动生成 SRT 格式字幕：

  • 精准的时间轴对齐
  • 标准的 SRT 格式输出
  • 可直接用于视频编辑软件

4. AI图像生成

Section titled “4. AI图像生成”

集成 DrawThings 实现场景图像生成：

  • 基于章节内容智能生成提示词
  • 使用本地 Stable Diffusion 模型
  • 每个章节生成多张场景图像
  • 支持风格自定义

5. 剪映项目导出

Section titled “5. 剪映项目导出”

生成可直接导入剪映的项目结构：

  • 一键打开草稿文件
  • 音频、字幕、图像自动对齐
  • 支持直接编辑和发布

6. MCP服务集成

Section titled “6. MCP服务集成”

实现 Model Context Protocol (MCP) 协议：

  • 与 Ollama Desktop 无缝集成
  • 丰富的工具集合供 AI 代理调用
  • 支持命令行和 Web 界面两种操作方式

7. Web控制台界面

Section titled “7. Web控制台界面”

提供直观的 Web 操作界面：

  • 上传小说文件
  • 选择处理工具
  • 实时查看处理进度
  • 浏览输出结果

常见问题

Section titled “常见问题”

Q1：支持哪些操作系统？

Section titled “Q1：支持哪些操作系统？”

A：目前主要支持 macOS，未来计划扩展到 Windows 和 Linux。

Q2：可以处理英文小说吗？

Section titled “Q2：可以处理英文小说吗？”

A：主要针对中文小说优化，英文支持取决于 TTS 模型的语言能力。

Q3：生成的视频质量如何？

Section titled “Q3：生成的视频质量如何？”

A：音频质量取决于 TTS 模型，图像质量取决于 Stable Diffusion 模型。使用默认模型可获得不错的效果。

Q4：需要付费吗？

Section titled “Q4：需要付费吗？”

A：novel-video-workflow 是开源免费的，但依赖的工具（如某些 SD 模型）可能有不同的许可证。

Q5：如何提高语音自然度？

Section titled “Q5：如何提高语音自然度？”

A：

  • 使用高质量参考音频
  • 调整 TTS 参数
  • 尝试不同的 TTS 模型
  • 适当添加标点符号控制节奏

Q6：处理速度如何？

Section titled “Q6：处理速度如何？”

A：取决于硬件配置和章节长度。Apple Silicon Mac 图像生成速度较快，一般章节可在几分钟内完成。

服务启动失败

Section titled “服务启动失败”

问题：程序停止执行并显示错误信息

解决方案：

  1. 检查 Ollama 服务是否运行：ps aux | grep ollama
  2. 检查 DrawThings 是否开启 HTTP 访问
  3. 检查 IndexTTS2 服务是否在 7860 端口
  4. 确认端口未被占用

图像生成失败

Section titled “图像生成失败”

问题：某些章节没有生成图像

解决方案：

  1. 检查 DrawThings 服务状态
  2. 查看章节内容是否足够生成提示词
  3. 检查磁盘空间
  4. 查看 logs 获取详细错误信息

音频质量问题

Section titled “音频质量问题”

问题：生成的音频有杂音或断续

解决方案：

  1. 更新 IndexTTS2 到最新版本
  2. 检查参考音频质量
  3. 调整采样率设置
  4. 尝试不同的 TTS 模型
