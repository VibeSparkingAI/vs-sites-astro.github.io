Google Antigravity 会写代码也会画图：IDE 里一键生成 UI 和配图

2025年12月5日
Vibe Sparking

直接说重点： Google Antigravity 这个新 IDE，不只会写代码，还能在编辑器里「顺手帮你画图」——UI 草图、Logo、营销图都能一键生成。

一、Antigravity 是什么鬼？为啥工程师都在聊

Section titled “一、Antigravity 是什么鬼？为啥工程师都在聊”

简单理解：

Antigravity = VS Code 外形 + 多智能体 AI 内核 + Gemini 3 Pro + 内置图片生成

它是 Google 推出的 AI 优先（agent-first）开发平台，在编辑器、终端、内置浏览器里跑一群 AI Agent，帮你干从规划、写码、测试到 Demo 的一整套活。([Google Developers Blog][1])

Antigravity 目前是 免费公测版，支持 Windows / macOS / Linux，用的是 Gemini 3 Pro 为主，也能接 Claude Sonnet 4.5 和一些开源模型。([Wikipedia][2])

对我们这种写前端、做产品 Demo、搞 side project 的人来说，它就是：

打开 IDE = 打开一个「带画师和小工的工作室」

二、Antigravity 里怎么「画图」？底层是啥模型

Section titled “二、Antigravity 里怎么「画图」？底层是啥模型”

1）内置两套图像模型：Imagen 3 + Nano Banana Pro

Section titled “1）内置两套图像模型：Imagen 3 + Nano Banana Pro”

按官方和社区的说法，现在 Antigravity 里的图片生成主要有两条线：([Google Antigravity][3])

  • 默认：Imagen 3 用来在 IDE 里生成插画、配图，偏通用创作。

  • 进阶：Nano Banana Pro（Gemini 3 Pro Image） 这是 Google 新推的图像模型，支持：

    • 更清晰的文字渲染（海报里的英文小字不再糊）
    • 更复杂的逻辑和布局（比如数据可视化、信息图）
    • 更高分辨率（1K / 2K / 4K 输出）
    • 支持多张参考图做风格 / 构图统一

官方博客和文档都写了：Nano Banana Pro 已经整合进 Antigravity，可以在 Antigravity 里直接调用它生成 UI Mockup、Logo、营销图等各种视觉素材。([Google Antigravity][3])

2）图片生成不是单独玩具，而是工作流的一环

Section titled “2）图片生成不是单独玩具，而是工作流的一环”

InfoWorld 的上手体验里专门提到：Antigravity 的 Agent 可以直接调用 Nano Banana，帮你做 界面 Mockup 和图像素材。([InfoWorld][4])

典型场景比如：

  • 为一个新页面生成几版 UI 草图
  • 为产品生成 Logo 备选方案
  • 为营销页生成插画 / 配图
  • 为文档自动配图、做示意图

有趣的是，有作者测试时，因为图像服务超负载，Agent 生成图片失败了一阵，它直接「换路子」，改成写了一整个 HTML/CSS 的界面来当 Mockup。这就是 Agent 工作流的味道：

“要么给你图，要么给你能跑的前端页面。”

三、对前端、产品和独立开发者有啥实际好处？

Section titled “三、对前端、产品和独立开发者有啥实际好处？”

1）「边写需求边出图」的开发体验

Section titled “1）「边写需求边出图」的开发体验”

真实用户在 Reddit 分享了一个例子：

做房车租赁网站，需要 6 张热门目的地的配图。 他直接在 Antigravity 里让 Agent 生成合适的照片，并自动应用到卡片组件里，大概 1 分钟搞定。([Reddit][5])

这跟我们传统流程完全不一样：

以前：写完页面 → 打开图库网站 / 画图工具 → 下载 / 导出 → 再塞回项目 现在：在 IDE 里一句话 → 图片 + 代码一起就位

对赶 MVP、黑客松、内部 Demo 的人来说，这相当于把「找图 / 画图」这块时间压缩到几分钟。

2）UI 设计和实现之间的墙被打薄了

Section titled “2）UI 设计和实现之间的墙被打薄了”

Antigravity 的 Agent 会把生成的图片、Mockup、代码改动，都整理成 Artifacts（成果物），你可以：([Google Antigravity][6])

  • 一边看 Mockup，一边看代码 diff
  • 在 Artifact 上直接评论：「按钮再大一点」「主色调换成蓝色」
  • 再让 Agent 按反馈改图、改代码

这对团队协作也挺友好：

  • 产品 / 设计：在 Artifact 页面用自然语言提意见
  • 开发：专心看 diff 和逻辑
  • Agent：负责在中间跑腿、改图、改代码

3）单人创业者：一个 IDE 顶一个小团队

Section titled “3）单人创业者：一个 IDE 顶一个小团队”

如果你是一个人做项目：

  • 文案：让模型写个 Landing Page 副本
  • 设计：让 Antigravity 生成插画、UI Mockup、Logo 草稿
  • 开发：复用同一个 Agent，把图塞进前端代码里，顺便帮你写 API / 后端

等于：

打开 Antigravity = 同时请了一个文案、一个画师、一个初级工程师 你只需要做「总监」和「拍板的人」

四、实际体验：在 Antigravity 里生成一张图，大概要走几步？

Section titled “四、实际体验：在 Antigravity 里生成一张图，大概要走几步？”

根据 Google 的官方 Codelab 和几篇教程，流程大概是这样：([Google Codelabs][7])

以下是「图 + 前端代码」的一种典型玩法，给你一个脑补画面。

步骤 1：开一个任务，让 Agent 帮你做页面和配图

Section titled “步骤 1：开一个任务，让 Agent 帮你做页面和配图”

在 Agent Manager 里新建一个 Task，比如：

「帮我做一个健身 App 的着陆页，包含 hero section，生成一张主视觉插画，用来放在页面顶部。」

Agent 会自动：

  • 规划任务（生成一个 Task Plan）
  • 决定需要哪些文件、哪些图
  • 在编辑器里创建 / 修改代码文件

步骤 2：调用图片模型生成视觉稿

Section titled “步骤 2：调用图片模型生成视觉稿”

在执行计划里，Agent 会调用内置的图片模型：

  • 默认用 Imagen 3
  • 如果你按教程配置了脚本，也可以指定走 Nano Banana Pro（Gemini 3 Pro Image），适合做更复杂、带文字的设计稿([Zenn][8])

它会在 Artifact 面板里给你一组图片候选：

  • 你能直接预览
  • 不喜欢就让它「regenerate with …」
  • 比如加一句「make it more minimal, more dark theme」

步骤 3：把图塞进代码里

Section titled “步骤 3：把图塞进代码里”

根据体验文章，多数人会选择两种方式：([Medium][9])

  1. 由 Agent 直接改代码

    • Agent 把图片保存到项目静态资源目录
    • 自动更新 React / Vue / Next.js 页面里的 <img> 或 Image 组件

  2. 你手动挑图 + 调 UI

    • 手动选中某张图
    • 自己微调布局、文案和样式

有些人甚至用它来做「图片优化工具的 UI + 配图」，完全当成一个图形产品的 AI 搭档来用。([Enchanter Consulting][10])

五、别忘了配额和安全坑：太爽也要踩刹车

Section titled “五、别忘了配额和安全坑：太爽也要踩刹车”

1）免费图像生成是有额度的

Section titled “1）免费图像生成是有额度的”

最近 Google 和 OpenAI 都在悄悄收紧免费额度：

  • Gemini 3 Pro 和 Nano Banana Pro 的免费调用次数被压缩
  • 免费用户每天能生成的图片张数不多，大概 2～3 张浮动([TechRadar][11])

这意味着：

  • Antigravity 里狂刷图一定会撞配额
  • 做产品 Demo / 内部汇报够用
  • 真要量产素材，可能就得上付费套餐或配合其它工具

2）IDE 里接 Agent，要对权限有点敬畏

Section titled “2）IDE 里接 Agent，要对权限有点敬畏”

安全研究团队已经指出，Antigravity 这种「Agent 自动跑终端命令、访问文件」的设计，本身也带来一些安全风险，比如：([TechRadar][12])

  • prompt injection 让 Agent 执行奇怪的命令
  • 不小心把 .env 或敏感日志读出来
  • 权限开太大时，甚至可能误删文件

跟图片本身无关，但是：

既然你都让它帮你改代码、传文件、跑脚本了，顺手检查一下权限设置值了。 图片生成功能很酷，但别顺手把服务器也“画”没了。

六、写在最后：IDE 里自带「画师」，你会用吗？

Section titled “六、写在最后：IDE 里自带「画师」，你会用吗？”

简单总结一句：

Antigravity 把「会写代码的 AI」升级成了「会写代码、会画图、会跑浏览器的一整队 Agent」。

对我们这类做产品、做前端、做独立开发的人来说，它主要有三层价值：

  • MVP 和 Demo 速度明显加快
  • UI / 视觉和代码之间，不再是两条断开的流水线
  • 一个人做项目，也能玩出「小团队」的工作流

你自己会在哪些场景里用 IDE 自带的图片生成功能？

  • 做公司内部评审的 Demo？
  • 做 side project 的 Landing Page？
  • 还是直接当低成本「设计搭子」？

欢迎在评论区聊聊你的想法，也可以转给身边的前端 / 产品 / 独立开发者一起试试这波新工具。
