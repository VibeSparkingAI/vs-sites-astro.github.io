Claude Code 生成 Flutter 多环境 Flavors 实战：一套代码，多个环境跑/构建

2025年9月17日
Vibe Sparking

想要在一个 Flutter 代码仓里同时维护开发版（dev）、预发布（staging/TestFlight）、生产版（prod）？最稳妥的方式就是用 flavors。

本文用一套最小可行的实践，带你用 Claude Code：

  • 跑起多个 flavor（dev/staging/prod）
  • Android/iOS 侧完成必要配置
  • 在 Dart 代码里依据 flavor 切换接口、日志、监控、图标等
  • 命令行与 CI 构建范式一把梭

——写完即可复制到日常项目里用。

Claude Code 提示词模板

Section titled “Claude Code 提示词模板”
  • Android productFlavors：在 Gradle 中声明不同环境（applicationId 后缀、资源替换、签名、图标）。
  • iOS Schemes + Build Configurations：为 dev/staging/prod 建独立 Scheme，并指向对应的 Build Configuration 与 Bundle ID。
  • Dart --dart-define：把环境变量注入到代码里（如 BASE_URL、SENTRY_DSN）。
  • 多入口文件：lib/main_dev.dart、lib/main_staging.dart、lib/main_prod.dart，各自初始化。

常见坑与排查

Section titled “常见坑与排查”
  • iOS 运行报找不到 Scheme：确认 Xcode 中的 Scheme 名与 --flavor 完全一致。
  • Android 构建找不到任务：assemble<Flavor><BuildType> 大小写要对（如 assembleDevDebug）。
  • 图标/名称无变化：检查是否使用了 @string/app_name，以及各 flavor 的 resValue 或资源目录是否生效。
  • AGP 8+ 报 namespace：在 android { namespace "..." } 指定 namespace。
  • 机密管理：dart-define 用在 CI 时避免把密钥写进仓库；本地 .env 不要入库。
