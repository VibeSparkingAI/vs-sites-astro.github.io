如何欺骗大脑去做困难的事情

Section titled “如何欺骗大脑去做困难的事情”

来源： @DrDominicNg | 发布： 2025年12月4日

引言

Section titled “引言”

我们都有过这种经历：盯着一个明知应该完成的任务，却不知为何无法迈出第一步。我们跳过的健身、不写的文章、永远学不会的语言。

作为一名神经科学家，Dr. Dominic Ng 每天都在观察这种挣扎——而解决方案不是更多的意志力。而是与大脑的自然趋势合作，而不是对抗它。

这里有 7 种经过验证的方法来欺骗大脑去做困难的事情。

1. 将任务缩减到仅两分钟

Section titled “1. 将任务缩减到仅两分钟”

问题： 你的大脑抗拒大的承诺。当你想到”锻炼一小时”时，你的杏仁核会发出警报。

解决方案： 不要做出大的承诺。做出一个微小的承诺。

  • “锻炼” → 做一个练习动作
  • “打扫房子” → 捡起三件物品
  • “写报告” → 打开文档并写一句话

两分钟法则之所以有效，是因为它绕过了大脑的威胁检测系统。一旦你开始，动量就会接管。

2. 在开始之前做好准备

Section titled “2. 在开始之前做好准备”

洞察： 不要开始任务——只是开始准备工作。

示例：

  • 穿上健身服
  • 打开笔记本电脑
  • 清理桌面

为什么有效： 身体活动绕过了精神阻力。在你真正”开始”实际任务之前，动量就已经启动了。

这种技巧利用了心理学家所说的”执行意图”——连接意图和行动的具体如果-那么计划。

3. 在困难任务前后夹上奖励

Section titled “3. 在困难任务前后夹上奖励”

模式：

关键： 让奖励取决于完成该任务。你的大脑会追逐它。

这种技巧建立在操作性条件反射之上——你的大脑学会将努力与随之而来的奖励联系起来。随着时间的推移，努力本身也会变得令人愉悦。

4. 在做任务时让它变得愉快

Section titled “4. 在做任务时让它变得愉快”

示例：

  • 在有氧运动时听你最喜欢的播客
  • 在深度工作时喝你最好的咖啡
  • 写作时点一支蜡烛

结果： 你的大脑会学习到：努力 = 享受。

这被称为”诱惑捆绑”——将你需要做的事情和你想要做的事情配对。快乐和努力的神经通路交织在一起，随时间减少阻力。

5. 将其框架化为实验，而非表演

Section titled “5. 将其框架化为实验，而非表演”

转变：

  • ❌ “我必须搞定这个演示文稿”
  • ✅ “我正在测试什么能获得更多参与”

为什么重要： 实验创造好奇心。表演创造压力。

当你将任务框架化为实验时，失败变成数据，而不是挫败。这激活了大脑的好奇心驱动学习系统，而不是威胁回避回路。

6. 像你想成为的人那样行动

Section titled “6. 像你想成为的人那样行动”

转变：

  • “我需要改善体形” → 惯性
  • “我是一个活跃的人” → 行动

科学： 身份驱动行为比意志力更有效。

《原子习惯》作者 James Clear 称之为”基于身份的习惯”。当你改变你相信自己的身份时，你的行为自然会与该身份保持一致。

7. 称自己为初学者

Section titled “7. 称自己为初学者”

重构：

  • ❌ “我不擅长语言”
  • ✅ “我正在学习西班牙语”

力量： 初学者无法失败。他们只能进步。

初学者心态激活以成长为导向的神经通路。当你在”学习”时，错误是预料之中的。当你”不擅长”时，错误会确认你的负面自我认知。

这背后的神经科学

Section titled “这背后的神经科学”

你的大脑本质上是一个节约能量的预测机器。困难的任务发出信号：

  1. 高能量消耗
  2. 结果的不确定性
  3. 失败的潜在可能

这些会触发你的杏仁核——大脑的威胁检测系统——然后启动回避行为。

上述七种技巧之所以有效，是因为它们：

  • 减少感知的威胁（微小任务，初学者心态）
  • 创造动量（准备阶段，两分钟法则）
  • 建立积极联想（奖励，享受捆绑）
  • 转移身份（“假装”行动，实验化）

关键要点

Section titled “关键要点”
技巧神经机制最适合
两分钟法则绕过杏仁核威胁反应开始任何困难的事情
先准备意识”开始”前的动量有准备障碍的任务
奖励三明治多巴胺驱动的强化建立新习惯
诱惑捆绑联想学习锻炼、家务、无聊任务
实验框架化好奇心 vs. 威胁回路表演焦虑、创造性工作
身份转移自我概念驱动的行为长期行为改变
初学者心态成长型心态激活学习新技能

日常应用：示例工作流程

Section titled “日常应用：示例工作流程”

以下是如何将这些技巧组合起来应对困难任务：

场景： 你需要写一份报告但感到卡住。

  1. 缩小它： “我会打开文档并写下标题”
  2. 准备： 清理桌面，拿水，打开笔记本电脑（奖励：最喜欢的播放列表）
  3. 捆绑： 在写初稿时听播客
  4. 实验： “我正在测试哪种结构更流畅”
  5. 身份： “我是一个清晰沟通的人”
  6. 初学者： “我正在学习更简洁地写作”

更深层的教训

Section titled “更深层的教训”

你的大脑不懒惰——它是高效的。它被设计为生存而保存能量。Dr. Ng 分享的技巧不与你的大脑天性对抗；它们与它合作。

最深刻的见解是：阻力不是性格缺陷。 它是你的大脑在完全做它进化来做的事情——保护你免受不确定性和不必要努力的影响。

通过理解并与这些机制合作，你不仅完成更多事情。你与自己的关系更友善，认识到开始挣扎不是个人失败——而是生物学。

这也许是终极的技巧：认识到你的大脑在你这一边，并学会用它的语言说话。

Agent Zero 有何不同？

Section titled “Agent Zero 有何不同？”

大多数 AI 框架都带有预编程的行为和硬编码的轨道。Agent Zero 采用了完全不同的方法：

  • 非预定义框架——设计为动态成长和学习
  • 完全透明——每个提示词、工具和行为都可读且可定制
  • 计算机作为工具——它根据需要编写自己的代码和工具
  • 多智能体层级——智能体创建下属来解决复杂任务
  • 持久记忆——解决方案和事实被记住以提高未来效率

Section titled “核心架构：Agent Zero 如何工作”

1. 通用助手理念

Section titled “1. 通用助手理念”

Agent Zero 不专注于单一任务——它被设计为你的通用个人助手。给它任何任务：

  • 开发项目——“创建一个具有实时数据可视化的 React 仪表板”
  • 数据分析——“分析上季度 NVIDIA 的销售数据并创建趋势报告”

  • 系统管理——“为我们的 Web 服务器设置监控”

Section titled “2. 工具创建理念”

默认工具：

  • 在线搜索
  • 记忆功能
  • 通信（用户和智能体之间）
  • 代码/终端执行

其他所有内容都由智能体自己创建。 需要特定的 API 集成？Agent Zero 编写代码。需要数据处理管道？它使用终端构建。

3. 多智能体协作

Section titled “3. 多智能体协作”

每个智能体都有一个提供任务和指令的上层智能体。这创建了一个自然的层级结构：

  • Agent 0 的上层= 人类用户
  • 下属智能体= 在任务需要分解时自动创建
  • 清晰的上下文= 每个智能体专注于其特定的子任务
  • 结果向上流动= 每个智能体向其上层报告

这种方法保持上下文专注，并支持并行问题解决。

4. 完全可定制

Section titled “4. 完全可定制”

没有什么被隐藏。一切都可以改变：

  • 系统提示词——prompts/default/agent.system.md 定义核心行为
  • 消息模板——每个通信模板都存在于 prompts/ 中
  • 工具——python/tools/ 中的默认工具可以被修改或复制
  • 扩展——通过扩展框架添加新功能

Section titled “项目：隔离的工作空间”

  • 专用文件存储

NautilusTrader：基于 Rust 和 Python 构建的高性能算法交易平台

简介

Section titled “简介”

NautilusTrader 是一个开源、高性能、生产级的算法交易平台，为量化交易者提供基于事件驱动引擎的历史数据回测能力，可用于回测自动交易策略组合，并且能够直接部署到实盘交易——无需修改任何代码。

在 GitHub 上拥有超过 17,000 颗星，并保持双周发布节奏，NautilusTrader 已在算法交易领域确立了重要地位。其独特之处在于 AI 优先的设计理念，解决了量化交易中的一个关键挑战：保持研究/回测环境与生产实盘交易系统的一致性。

核心特性与亮点

Section titled “核心特性与亮点”

架构设计：Rust 核心 + Python 原生

Section titled “架构设计：Rust 核心 + Python 原生”

NautilusTrader 采用混合架构，充分利用 Rust 和 Python 的优势：

  • Rust 核心：性能关键组件使用 Rust 编写，通过 tokio 实现异步网络通信
  • Python 原生环境：交易策略使用 Python 开发，通过 CPython C 扩展模块集成
  • 零代码变更：同一套策略代码既可用于回测，也可直接用于实盘交易

这种架构消除了传统工作流程中策略研究时使用 Python 向量化方法，然后在生产环境中需要用 C++、C# 或 Java 重新实现的问题。

性能与可靠性

Section titled “性能与可靠性”
  • 快速：Rust 驱动的核心组件，通过 tokio 实现异步网络通信
  • 可靠：Rust 保证的类型和线程安全，可选的 Redis 状态持久化支持
  • 便携：操作系统无关，支持 Linux、macOS 和 Windows，可通过 Docker 部署
  • 纳秒精度：支持纳秒级分辨率的历史数据，确保回测准确性

高级交易功能

Section titled “高级交易功能”
  • 订单类型：支持 IOC（立即成交或取消）、FOK（全部成交或取消）、GTC（一直有效）、GTD（日期有效）、DAY（当日有效）、AT_THE_OPEN（开盘时）、AT_THE_CLOSE（收盘时）等时效指令
  • 高级订单：条件触发器、执行指令（仅做市商、仅减仓、冰山订单）
  • ** contingent 订单**：OCO（一方成交另一方取消）、OUO（一方成交另一方更新）、OTO（一方成交触发另一方）
  • 多交易所：同时在多个交易所交易，支持做市和套利策略
  • 资产类别无关：外汇、股票、期货、期权、加密货币、DeFi，甚至博彩交易所

AI 训练就绪

Section titled “AI 训练就绪”

回测引擎经过优化，速度足以支持使用强化学习（RL）和进化策略（ES）训练 AI 交易代理，可直接在平台内进行机器学习工作流。
