Vibe Sparking

引言

Section titled “引言”

2026 年，随着 AI 智能体和 Claude Code 的普及，越来越多的开发团队开始为 AI 助手创建专门的”技能包”。Expo 团队率先发布了官方的 expo/skills 仓库——这是一个面向 Expo 和 React Native 开发的 AI 智能体技能集合。

什么是 Expo Skills？

Section titled “什么是 Expo Skills？”

expo/skills 是 Expo 团队官方维护的 AI 智能体技能仓库，专为构建、部署和调试 Expo 应用而设计。这些技能经过 Expo 团队内部使用验证，针对 Claude Opus 模型进行了优化，但也兼容任何 AI 智能体。

项目概况：

  • 仓库地址：https://github.com/expo/skills
  • Star 数：149+
  • Fork 数：6+
  • 许可证：MIT
  • 主要语言：JavaScript

Section titled “安装方式”

Claude Code 用户（推荐）

Section titled “Claude Code 用户（推荐）”

任何 AI 智能体

Section titled “任何 AI 智能体”

Section titled “核心技能插件”

1. Expo App Design（应用设计）

Section titled “1. Expo App Design（应用设计）”

功能概览：

  • 遵循 Apple Human Interface Guidelines 的 UI 设计指南
  • Expo Router 导航模式（栈、标签、模态、 sheets）
  • 原生 iOS 控件、SF Symbols、动画和视觉效果
  • EAS Hosting 的 API 路由创建
  • React Query 数据获取和离线支持
  • Tailwind CSS v4 与 NativeWind v5 配置
  • DOM 组件集成（在原生应用中运行 Web 代码）

适用场景：

  • 从零开始构建新的 Expo 应用
  • 添加导航、样式或动画
  • 设置 API 路由或数据获取
  • 通过 DOM 组件集成 Web 库

包含技能：

  • building-ui - UI 组件、导航、样式和原生控件
  • api-routes - EAS Hosting 服务端 API 路由
  • data-fetching - 网络请求、缓存和离线支持
  • dev-client - 开发构建和 TestFlight 分发
  • tailwind-setup - Tailwind CSS v4 配置
  • use-dom - 通过 DOM 组件集成 Web 代码

2. Upgrading Expo（SDK 升级）

Section titled “2. Upgrading Expo（SDK 升级）”

功能概览：

  • 引导完成分步升级流程（npx expo install expo@latest）
  • 识别已弃用包及其现代替代品
  • 清理过时的配置（babel、metro、postcss）
  • 处理托管和裸工作流的缓存清理
  • 了解 SDK 版本间的重大变更

适用场景：

  • 升级到新的 Expo SDK 版本
  • 修复升级后的依赖冲突
  • 从已弃用包迁移（如 expo-av → expo-audio/expo-video）
  • 清理遗留配置文件

3. Expo Deployment（应用部署）

Section titled “3. Expo Deployment（应用部署）”

功能概览：

  • 指导 iOS App Store 和 TestFlight 提交流程
  • 涵盖 Android Play Store 部署和轨道管理
  • 解释 EAS Build 配置和版本管理
  • 帮助编写和验证 EAS Workflow YAML 文件（CI/CD）
  • EAS Hosting 的 Web 部署

适用场景：

  • 向 App Store Connect 或 Google Play 提交应用
  • 设置 TestFlight Beta 测试
  • 配置 EAS Build 配置文件
  • 编写自动部署的 CI/CD 工作流
  • 使用 EAS Hosting 部署 Web 应用

包含技能：

  • deployment - 应用商店提交、EAS Build 和 Web 托管
  • cicd-workflows - CI/CD 自动化的 EAS Workflow YAML 文件

Section titled “4. EAS CI/CD Workflows”

持续集成和持续部署的自动化工作流配置技能，帮助开发者设置完整的自动化部署管道。

什么是 SkillMaster？

Section titled “什么是 SkillMaster？”

SkillMaster.cc 是一个精选市场，托管适用于多个 AI 编码平台的生产级技能包。与 GitHub 仓库需要手动集成不同，SkillMaster 提供：

  • 经过验证的技能：每个技能包都经过安全和质量审计
  • 跨平台兼容性：技能可跨 Claude Code、Codex CLI 和 ChatGPT 使用

  • 版本控制：跟踪更新和兼容性说明

Section titled “智能体技能革命”

  1. 发现 特定领域的预构建技能
  2. 安装 直接安装到 AI 助手
  3. 调用 使用简单命令调用技能

Docker 收购 MCP Defender：AI 智能体一直缺失的安全防护层

随着 AI 智能体在开发工作流中变得越来越自主，安全形势已经发生了根本性的变化。MCP Defender 应运而生——这款桌面应用最近被 Docker 收购，用于保护 Cursor、Claude Desktop、VS Code 和 Windsurf 等 AI 应用免受各种攻击。

1. MCP Defender 是什么？

Section titled “1. MCP Defender 是什么？”

MCP Defender 是一款桌面应用程序，作为模型上下文协议（MCP）的安全代理。它的工作原理如下：

  • 🛡️ 自动保护：所有 AI 应用的 MCP 工具调用请求和响应都会自动通过 MCP Defender 代理
  • 🔎 签名检测：拦截的数据会与安全签名库进行比对
  • 🔐 用户可控拦截：当检测到有害内容时，MCP Defender 会发出警告并询问是否允许或阻止该工具调用

支持的 AI 应用：

  • Cursor
  • Claude Desktop
  • Visual Studio Code
  • Windsurf

2. Docker 为何收购 MCP Defender

Section titled “2. Docker 为何收购 MCP Defender”

2025 年 9 月 5 日，Docker 宣布收购 MCP Defender，这标志着其在 AI 智能体安全领域的战略布局。理由很明确：

“AI 智能体正变得越来越智能——也越来越自主。这意味着新的风险、新的攻击面，以及对能够跟上开发团队速度的运行时安全的需求。“

A2UI：Google 开源的 Agent 驱动界面协议，改变一切

2025年12月21日

从纯文本响应到丰富原生界面：Google A2UI 协议如何让 AI Agent 生成交互式用户界面

1. 功能亮点介绍

Section titled “1. 功能亮点介绍”

什么是 A2UI？

Section titled “什么是 A2UI？”

2025年12月15日，Google 正式公开发布了 A2UI（Agent to UI）——一个开源协议，让 AI 智能体能够生成丰富的交互式用户界面，在各平台上以原生方式渲染，无需执行任意代码。

想象一下：AI 智能体不再只是返回纯文本，而是可以发送一个完整的交互式表单、数据可视化卡片或预订确认界面。神奇之处在于，你的应用使用原生组件来渲染这些界面，既保持了你的设计系统，又享受了 AI 生成界面的便利。

解决的核心问题

Section titled “解决的核心问题”

传统智能体交互面临根本性的矛盾：

方案问题
纯文本响应表达能力有限，来回对话繁琐
iframe 中的 HTML/JavaScript安全风险、沙箱复杂、无法融入原生样式
A2UI 方案声明式 JSON 数据 + 原生渲染 = 两全其美

核心特性

Section titled “核心特性”
特性描述
安全优先声明式数据格式（非可执行代码）。智能体只能渲染目录中预批准的组件
LLM 友好扁平化组件列表加 ID 引用，便于 LLM 增量生成
框架无关适用于 React、Flutter、Angular、SwiftUI、Lit 等
传输无关兼容 A2A、AG-UI、REST（实验性）
渐进式渲染用户可实时看到 UI 随 LLM 生成而构建

重要注意事项

Section titled “重要注意事项”
领域建议
安全性目录中只包含你信任的组件。A2UI 是声明式的而非可执行的，但你的渲染器实现必须安全
版本控制A2UI 当前为 v0.8（公开预览版）。v1.0 之前预期会有破坏性变更
性能对于复杂 UI，考虑在渲染器中实现分页和懒加载
无障碍性你的渲染器负责无障碍功能。A2UI 提供结构；你提供语义

常见问题

Section titled “常见问题”

问：A2UI 与发送 HTML 有什么不同？

答：A2UI 发送描述 UI 组件的结构化 JSON。你的应用使用原生组件来渲染，保持你的设计系统、无障碍性和安全性。HTML/JS 需要沙箱且无法与原生样式集成。

问：智能体能通过 A2UI 执行任意代码吗？

答：不能。A2UI 是声明式数据格式。智能体只能请求预批准目录中的组件。你的渲染器决定如何呈现它们。

问：A2UI 会取代 MCP（Model Context Protocol）吗？

答：不会，它们是互补的。MCP 将智能体连接到工具和数据。A2UI 定义智能体如何描述 UI。你可以在同一系统中同时使用两者。

问：AG-UI 呢？

答：AG-UI 是实时智能体-UI 通信的传输层。A2UI 定义载荷格式。它们完美协作。

问：A2UI 可以用于生产环境吗？

答：A2UI 是 v0.8（公开预览版）。功能可用但仍在演进中。生产使用应规划规范更新。

结语

Section titled “结语”

A2UI 代表了 AI 智能体与用户交互方式的根本性转变。智能体不再只能返回纯文本或冒险执行代码，现在可以描述丰富的交互式 UI，在各平台上原生渲染。

核心要点：

  4. 可组合 —— 与 MCP、AG-UI、A2A 协同工作，构建完整的智能体系统

随着智能体时代的加速发展，A2UI 这样的协议将成为构建原生、安全、响应式 AI 应用的关键基础设施。
