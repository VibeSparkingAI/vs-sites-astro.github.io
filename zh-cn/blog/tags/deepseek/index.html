Alma：一款优雅的桌面应用，统一管理你的所有 AI 服务

2025年12月25日
Vibe Sparking

AI 服务商的”多开”困境

Section titled “AI 服务商的”多开”困境”

如果你是 AI 深度用户，你的日常工作流程可能是这样的：

  • 一个 ChatGPT 标签页用于快速对话
  • 一个 Claude 标签页专门写代码和分析
  • 一个 Gemini 标签页处理 Google 生态任务
  • 一个 DeepSeek 标签页用于性价比场景
  • 散落在各处的 API 密钥，越来越多…

是不是很熟悉？ 你不是一个人。管理多个 AI 服务商，已经成为新时代的”浏览器标签页焦虑症”。

Alma 就是为解决这个问题而生的。

Alma 是什么？

Section titled “Alma 是什么？”

Alma 是一款设计精美的 macOS 桌面应用，充当你 AI 能力的统一指挥中心。不用再在各个网页界面之间来回切换，你可以在一个简洁直观的界面中访问所有 AI 服务商。

把它想象成 AI 的万能遥控器 —— 一个界面，多个大脑。

谁做的？

Section titled “谁做的？”

Alma 出自 yetone 之手，他正是 OpenAI Translator 的开发者 —— 这个项目在 GitHub 上斩获了 23100+ 星标。那个项目从一个 Chrome 插件进化成了多平台桌面应用，充分证明了 yetone 交付高质量 AI 驱动桌面应用的能力。

这个背景很重要。Alma 不是一个周末小项目 —— 它来自一个有着丰富经验、能够打造人们每天实际使用的工具的开发者。

核心功能

Section titled “核心功能”

统一聊天界面

Section titled “统一聊天界面”

简洁直观的聊天界面，将你所有的 AI 对话整合在一起：

  • Markdown 渲染，响应格式精美
  • 代码高亮，具备语法感知能力
  • 实时流式输出，生成过程即时呈现
  • 深色/浅色主题切换，采用 View Transitions API 实现丝滑动画

智能记忆系统

Section titled “智能记忆系统”

Alma 能够跨对话记住上下文：

  • 用户偏好在会话间持久保存
  • 对话上下文向前延续
  • 专门的记忆管理面板便于组织
  • 可查看、整理、清理记忆

网页搜索集成

Section titled “网页搜索集成”

内置实时信息获取能力：

  • AI 回答时可检索实时网页数据
  • 无需手动粘贴 URL 或上下文
  • 基于真实网页数据提供当前、准确的答案

高级工具调用

Section titled “高级工具调用”

不仅是聊天，Alma 还能让 AI 执行复杂任务：

  • 数学计算
  • 数据分析操作
  • 智能工具集成

支持的 AI 服务商

Section titled “支持的 AI 服务商”
服务商状态
OpenAI (GPT-4 等)已支持
Anthropic (Claude)已支持
Google Gemini已支持
DeepSeek已支持
自定义 API已支持

这种服务商无关的设计意味着你永远不会被锁定。根据任务需要选择模型 —— 创意写作用 GPT-4，分析用 Claude，控制成本用 DeepSeek。

平台支持

Section titled “平台支持”

目前可用：

  • macOS Apple Silicon (.dmg)
  • macOS Intel (.dmg)

即将推出：

  • Windows
  • Linux

注意：目前只有 macOS 版本经过测试并保证可用。Windows 和 Linux 支持正在开发中。

为什么选择 Alma？

Section titled “为什么选择 Alma？”

1. 服务商灵活性

Section titled “1. 服务商灵活性”

不同模型擅长不同任务。Claude 可能是你写代码的首选，而 GPT-4 在创意写作方面表现更佳。Alma 让你无需切换应用就能切换上下文。

2. API 成本可控

Section titled “2. API 成本可控”

使用自己的 API 密钥，成本完全由你掌控。没有中间商加价，没有订阅费意外。

3. 隐私优先设计

Section titled “3. 隐私优先设计”

作为使用你自己 API 密钥的桌面应用，你的对话不会经过第三方服务。数据只在你和选定的服务商之间流转。

4. 记忆连续性

Section titled “4. 记忆连续性”

与网页界面容易丢失上下文不同，Alma 的记忆系统在会话间保持连续性 —— 让它更像一个持久的助手，而不只是一个工具。

快速开始

Section titled “快速开始”
  1. 访问 alma.now
  2. 下载对应的 macOS 版本（Apple Silicon 或 Intel）
  3. 配置你偏好服务商的 API 密钥
  4. 开始聊天，统一访问多个 AI 模型

总结

Section titled “总结”

Alma 代表着一种转变：从 分散的 AI 网页服务 到 统一的 AI 桌面基础设施。

对于经常在多个 AI 服务商之间切换的用户来说，这种整合不仅是方便 —— 而是工作流程的变革。一个应用，多个模型，持久记忆，以及专为日常使用设计的界面。

如果你厌倦了在浏览器标签页之间来回切换，想要一种更精致的 AI 服务商管理方式，Alma 值得在你的 Dock 栏占有一席之地。

给 Claude 装上「长期记忆」：claude-mem 实战体验与使用建议

2025年12月9日
Vibe Sparking

claude-mem 这玩意儿，能让 Claude 真正「记住你」

你有没有这种感受：

每次打开 Claude / ChatGPT 写代码， 都要从头给它讲一遍项目背景：

「我在做个后台管理系统，技术栈是……」 「上次我们做到用户登录，这次要加的是……」

讲着讲着，人都烦了。

模型不是真的笨，只是没记忆。

今天这篇，就带你看看 GitHub 上一个很有意思的小项目： claude-mem——给 Claude Code 装上「长期记忆」的大脑。

一、claude-mem 到底干嘛的？

Section titled “一、claude-mem 到底干嘛的？”

一句话版：

它会自动记录你在 Claude Code 里的操作，自己总结、压缩， 下次再开同一个项目，Claude 会「主动想起」之前的上下文。

更细一点说，它是：

  • 一个 Claude Code 插件
  • 会自动捕获：你的指令、工具调用、读写文件等操作
  • 后台用 Claude 的 Agent SDK 做总结、提炼要点

下一次 Claude 要查历史，就不用「全量翻旧账」， 而是通过智能搜索 + 分层展开（progressive disclosure） 把少量但关键的上下文塞回对话中。

Q：我在 REPL 里还会看到“Claude”的字样，这是 DeepSeek 在跑吗？ A：是的。壳是 Claude Code，引擎是 DeepSeek。请求会按你设置的 ANTHROPIC_BASE_URL 与鉴权走 DeepSeek。

Q：我需要换 SDK 吗？ A：不用。如果你用的是 Claude Code 交互式工作流，只改环境变量即可。若在代码里直接调 SDK，可以用 Anthropic 官方 SDK，同时把 BASE_URL 指向 DeepSeek 的 Anthropic 端点即可。

Q：DeepSeek 兼容层有没有不一样的地方？ A：有。DeepSeek 列出了忽略/不支持的一些头与消息类型，工具大多支持，但图像/文档类消息暂不支持，注意规避。
