  • @mcp:github — 启用/禁用 MCP 服务器

在 git 仓库中，文件建议速度约快 3 倍，并支持模糊匹配。@ 是从”我需要上下文”到”Claude 有上下文”的最快路径。

规划：始终从计划开始

Section titled “规划：始终从计划开始”

花五分钟规划，能节省十分钟的审查时间。

无论任务大小，都要从计划开始。无论你的代理是否有内置的”计划模式”，你都可以直接要求：

“先制定一个计划，在做出更改之前与我确认。”

规划可能会感觉像是一种开销，但它将不可避免的工作提前了。审查计划比审查代码差异要快得多。

规划决定架构形态

Section titled “规划决定架构形态”

代理的目标是以最少的编辑次数让代码工作起来。一旦某个版本存在，后续的更改只会修补已有的内容，而不是重新思考方法。计划是你有机会在代码围绕错误架构固化之前选择正确形态的机会。

异步工作流：同时处理多个任务

Section titled “异步工作流：同时处理多个任务”

与编码代理异步工作（有时称为”后台运行”）让你能够：

  • 同时处理多个任务线程
  • 避免陷入拖延
  • 让代理持续运行较长时间，而你处理其他事务

典型工作流：让代理持续运行，同时你处理其他任务，定期检查进度。

使用YOLO模式实现真正的异步

Section titled “使用YOLO模式实现真正的异步”

只有在你不批准每个工具调用的情况下，异步才能发挥作用。要么使用代理的YOLO模式，要么给它一个明确的命令允许列表，覆盖它需要的几乎所有操作。

否则你就重新发明了结对编程——而且更慢。

不同平台的命名不同：‘yolo’、‘危险地跳过权限’或类似名称。

让代码库可以进行质量验证

Section titled “让代码库可以进行质量验证”

如果代理无法测试自己的更改，它就会要求你来做——这不是对你时间的很好利用。

**投入精力让更改可以通过开发或测试命令进行验证。**并非所有内容都可以由代理测试（前端仍然需要人眼查看），但大多数更改应该以”测试通过”结束，而不是”你能试试这个吗？“

解决开发服务器问题

Section titled “解决开发服务器问题”

代理不擅长启动和清理开发服务器，这会导致：

  • 端口冲突
  • 浪费时间排查”该端口上已有内容在运行”的错误

像 Dev Manager MCP 这样的解决方案可以自动分配端口并终止僵死的服务器。

添加虚拟数据以支持离线测试

Section titled “添加虚拟数据以支持离线测试”

使用真实的种子数据让项目易于离线运行。如果代理花费一半时间让应用进入可测试状态，你将得到：

  • 不稳定的结果
  • 更慢的进度

虚拟数据还能支持并行代理——如果它们共享真实数据库，一个可能使用与另一个不同的架构版本。

将此添加到你的系统提示词中：

我们想要尽可能简单的更改。我们不关心迁移。代码可读性最重要，我们愿意做出更大的更改来实现它。

先决条件

Section titled “先决条件”
  • 操作系统：Windows 10/11
  • 运行时：Node.js (v18+)
  • 核心工具：全局安装的 Claude Code CLI (npm install -g @anthropic-ai/claude-code)

安装步骤

Section titled “安装步骤”

  1. 克隆仓库

  2. 安装依赖 你需要为根目录、后端和前端分别安装依赖。

  3. 启动应用

    • 选项 1 (Windows)：直接运行 start.bat。
    • 选项 2 (手动)：在不同的终端中分别运行 npm run dev:backend 和 npm run dev:frontend。

  4. 访问：打开浏览器并访问 http://localhost:5173。

并发开发

Section titled “并发开发”

想象一下，你正在开发一个功能，但同时也需要修复一个紧急 Bug 并审查通过一个 PR。与其打开三个终端并迷失在上下文中，不如在 Claude Code Board 上创建三张卡片：“功能 X”、“Bug Y”和“PR 审查 Z”。你可以轻松地在它们之间切换并查看进度。

基于 Agent 的工作流

Section titled “基于 Agent 的工作流”

你可以通过在 agents 目录中创建 .md 文件来配置专门的 Agent（例如，“架构师”Agent 或“调试器”Agent）。在开始会话时，你可以选择最适合该任务的特定 Agent。

注意事项

Section titled “注意事项”
  • 安全性：此工具仅设计用于 本地/个人使用。它没有企业级的安全性（加密/认证），不应部署到公共网络。
  • 平台：目前针对 Windows 进行了优化。
