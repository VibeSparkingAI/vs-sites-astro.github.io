OpenClaw Telegram Topics: 一个群组运行多条并行任务流

Section titled “OpenClaw Telegram Topics: 一个群组运行多条并行任务流”

OpenClaw 是一个开源 AI 助手，支持 Telegram、Discord 等多个平台。它的”并发任务引擎”能力在配合 Telegram Topics 功能时效果尤为出色。本指南将带你在单个群组中配置多条独立任务流。

理解 OpenClaw 的并发模型

Section titled “理解 OpenClaw 的并发模型”

OpenClaw 通过 Sessions 处理任务：

  • 一条 Session = 一条独立任务流，拥有独立上下文
  • 多个 Session = OpenClaw 同时处理多条任务，互不干扰

在 Telegram 里，每个 Topic 就是一个 Session。一个群可以包含聊天、工作、资讯三个 Topic，每个都维护独立的对话和上下文。

Session Key 架构

Section titled “Session Key 架构”

OpenClaw 根据聊天位置生成 session key：

位置Session Key 格式
私聊共享主会话
群组agent:<agentId>:telegram:group:<chatId>
Topicagent:<agentId>:telegram:group:<chatId>:topic:<threadId>

:topic:<threadId> 后缀隔离了每个 Topic 的上下文。Topic A 的消息不会串到 Topic B。

Section titled “设置并行任务通道”

推荐的 Topic 结构

Section titled “推荐的 Topic 结构”

从三个 Topic 开始，覆盖常见工作流：

Topic用途使用场景
Chat日常聊天随意提问、快速问答
Work办正事文档处理、数据分析
Feed资讯巡逻新闻聚合、监控追踪

第一步：让 Bot 能看到群消息

Section titled “第一步：让 Bot 能看到群消息”

Telegram 的 Privacy Mode 默认阻止 bot 看到群消息。解决方法：

  1. 打开 @BotFather
  2. 发送 /setprivacy
  3. 选择你的 bot
  4. 选择 Disable
  5. 把 bot 从群里踢掉再重新拉进来
  6. 授予管理员权限

管理员身份可以确保 bot 看到所有消息，不受隐私设置影响。

第二步：在群里开启 Topics

Section titled “第二步：在群里开启 Topics”
  1. 打开群设置
  2. 找到 Topics 或 论坛 模式
  3. 开启它
  4. 创建 Chat、Work、Feed 三个 Topic

Section titled “配置选项说明”

测试你的配置

Section titled “测试你的配置”

快速验收清单：

  1. 在 Chat Topic 发消息：bot 应该只在被 @时回复（如果 requireMention: true）
  2. 在 Work Topic 发消息：bot 应该回复所有消息（如果 requireMention: false）
  3. 检查隔离性：用 /new 重置一个 Topic 的会话，其他 Topic 应该保持上下文

常见问题排查

Section titled “常见问题排查”

Bot 在群里完全没反应

Section titled “Bot 在群里完全没反应”

原因：Privacy Mode 还开着，或者 bot 没有管理员权限。

解决：在 @BotFather 重新运行 /setprivacy → Disable。确保 bot 是群管理员。

私聊正常但群里不正常

Section titled “私聊正常但群里不正常”

原因：隐私设置和 requireMention 配置的组合问题。

解决：

  1. 确认管理员身份
  2. 检查 requireMention 值是否符合你的意图
  3. 先试试显式 @bot 确认它能收到消息

回复出现在错误的 Topic

Section titled “回复出现在错误的 Topic”

原因：JSON 里的 Topic ID 配错了。

解决：用 @userinfobot 确认准确的 Topic thread ID，更新配置。

重置会话影响多个 Topic

Section titled “重置会话影响多个 Topic”

已知问题：在多 agent 配置中，对非默认 agent 使用 /new 可能同时重置它和默认 agent 的会话。

临时方案：每个群使用单 agent 配置，或关注 GitHub issue 等待修复。

最佳实践

Section titled “最佳实践”
  1. 从简单开始：在工作流稳定之前，最多用三个 Topic
  2. Topic 命名：清晰的名称帮助人和 bot 都理解上下文
  3. 选择性 @：高流量 Topic 保持 requireMention: true 避免刷屏
  4. 历史记录限制：根据 Topic 的上下文需求调整 historyLimit

Vercel Agent Skills：AI 编程助手的官方技能集

2026年1月28日
Vibe Sparking

AI 编程助手越来越强了，但怎么让它们懂更多的”行规”？Vercel Labs 最近放出了一个技能集合，把多年的工程经验打包成了 AI 能读懂的格式。

vercel-labs/agent-skills 这个仓库包含了五个生产级别的技能：React 优化、Web 设计规范、React Native 模式、React 组合模式，还有一键部署。

什么是 Agent Skills？

Section titled “什么是 Agent Skills？”

Agent Skills 就是给 AI 编程助手的”插件包”。跟普通文档不一样，这些技能是专门为 LLM 设计的：

  • AI 能自己读懂规则
  • 写代码或审查代码时自动应用
  • 识别上下文并激活对应技能
  • 需要的时候还能跑脚本

Vercel 的技能用的是 Agent Skills 格式，所以能在各种 AI 编程平台上用。

有哪些技能？

Section titled “有哪些技能？”

1. React 最佳实践

Section titled “1. React 最佳实践”

react-best-practices 收录了 40 多条规则，分 8 个大类，是 Vercel 工程团队十年来搞 React 和 Next.js 积累的经验。

什么时候用：

  • 写新的 React 组件或 Next.js 页面
  • 实现数据获取（客户端或服务端）
  • Code Review 找性能问题
  • 优化包大小或加载时间

包括：

  • 消除瀑布请求（关键） - 并行数据获取、预取
  • 包体积优化（关键） - Tree shaking、代码分割、动态导入
  • 服务端性能（高） - 服务端组件、流式渲染、缓存
  • 客户端数据获取（中高） - SWR/React Query 模式
  • 重渲染优化（中） - Memoization、useMemo、useCallback
  • 渲染性能（中） - 虚拟列表、CSS containment
  • JavaScript 微优化（低中） - 事件处理、属性访问

2. Web 设计指南

Section titled “2. Web 设计指南”

web-design-guidelines 能审查 UI 代码，100 多条规则覆盖了可访问性、性能和 UX 最佳实践。

什么时候用：

  • “帮我审查 UI”
  • “检查可访问性”
  • “审计设计”
  • “Review UX”
  • “按最佳实践检查我的网站”

包括：

  • 可访问性 - ARIA 标签、语义化 HTML、键盘导航
  • 焦点状态 - 可见的焦点指示器、focus-visible 模式
  • 表单 - 自动完成、验证、错误处理
  • 动画 - prefers-reduced-motion、合成器友好的 transform
  • 排版 - 弯引号、省略号、等宽数字
  • 图片 - 尺寸、懒加载、alt 文本
  • 性能 - 虚拟化、避免布局抖动、preconnect
  • 导航与状态 - URL 反映状态、深度链接
  • 深色模式与主题 - color-scheme、theme-color meta
  • 触摸与交互 - touch-action、点击高亮
  • 国际化 - Intl.DateTimeFormat、Intl.NumberFormat

3. React Native 指南

Section titled “3. React Native 指南”

react-native-guidelines 包含 16 条规则，覆盖移动端性能、架构和平台特定模式。

什么时候用：

  • 开发 React Native 或 Expo 应用
  • 优化移动端性能
  • 实现动画或手势
  • 使用原生模块或平台 API

包括：

  • 性能（关键） - FlashList、memoization、密集计算卸载
  • 布局（高） - Flex 模式、安全区域、键盘处理
  • 动画（高） - Reanimated 库、手势处理
  • 图片（中） - expo-image、缓存、懒加载
  • 状态管理（中） - Zustand 模式、React Compiler
  • 架构（中） - Monorepo 结构、import 优化
  • 平台（中） - iOS/Android 特定模式

4. 组合模式

Section titled “4. 组合模式”

composition-patterns 帮你避免布尔 prop 爆炸，通过复合组件、状态提升和内部组合来构建更灵活的 API。

什么时候用：

  • 重构布尔 prop 太多的组件
  • 构建可复用的组件库
  • 设计灵活的 API
  • Review 组件架构

包括：

  • 提取复合组件
  • 提升状态减少 prop 传递
  • 组合内部实现提升灵活性
  • 构建可组合的 API

Section titled “5. Vercel Deploy Claimable”

vercel-deploy-claimable 让你能直接在 AI 对话中把应用部署到 Vercel。部署是”可认领”的——用户可以把所有权转移到自己的 Vercel 账号。

什么时候用：

  • “帮我部署应用”
  • “部署到生产环境”

功能：

  • 从 package.json 自动检测 40+ 种框架
