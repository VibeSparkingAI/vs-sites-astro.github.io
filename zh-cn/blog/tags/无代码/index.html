Agentic Signal：本地 LLM 集成的可视化 AI 工作流自动化平台

2025年12月29日
Vibe Sparking

深入探索 Code Forge Temple 开发的 Agentic Signal — 一个可视化的拖拽式 AI 工作流自动化平台，通过 Ollama 集成在本地运行所有功能。无需编写代码即可构建智能工作流，无需依赖云服务。

简介

Section titled “简介”

Agentic Signal 是由 Code Forge Temple 开发的开源可视化 AI 工作流自动化平台。它使用拖拽式界面让用户能够构建由 AI 驱动的智能工作流，同时所有操作都在本地机器上运行。

Agentic Signal 的独特之处在于其对隐私优先的本地执行的承诺。通过与 Ollama 集成，它允许您利用大语言模型的力量，而无需将数据发送到云端。这使得它非常适合处理敏感文档、构建个人自动化工作流，或者只是希望完全掌控 AI 交互的场景。

仓库地址： code-forge-temple/agentic-signal 官方网站： agentic-signal.com

核心特性

Section titled “核心特性”
特性描述
可视化拖拽界面使用 React Flow 可视化构建工作流 — 无需编码
节点化架构无缝连接数据源、AI 处理器和输出节点
实时执行通过实时数据流可视化查看工作流运行
Ollama 集成在本地运行 AI 模型，保护隐私并提供完全控制
工具调用AI 代理可以执行函数并访问外部 API
结构化输出JSON schema 验证确保可靠的 AI 响应
对话记忆跨工作流执行维护上下文
浏览器自动化通过 Google Chrome 集成实现网页抓取和自动化

快速开始

Section titled “快速开始”

安装

Section titled “安装”

Agentic Signal 为 Windows、macOS 和 Linux 提供了预构建版本：

  1. 下载发布包 — 从 GitHub Releases 下载适合您操作系统的版本

  2. 安装 Google Chrome — 浏览器自动化和网页抓取节点需要

  3. 安装并运行 Agentic Signal

    • 运行安装程序并按照提示操作
    • Windows 用户： 首次启动时，杀毒软件可能会扫描应用程序（安装程序未签名）

  4. 配置 Ollama

    • 确保 Ollama 启用了 CORS 运行（桌面应用程序需要）
    • 在应用程序设置中配置 Ollama 主机地址

前置要求

Section titled “前置要求”

必需项

Section titled “必需项”

  • Google Chrome — 浏览器自动化和网页抓取节点需要

    • 从 google.com/chrome 下载

  • Ollama — 本地 LLM 运行时

可选项（用于 Google 服务集成）

Section titled “可选项（用于 Google 服务集成）”

如果要使用 Gmail、Google Drive 或 Google Calendar 节点，需要设置 OAuth2 凭证：

  1. 前往 Google Cloud Console
  2. 创建新项目或选择现有项目
  3. 启用要使用的 API（Gmail、Drive、Calendar）
  4. 创建 OAuth 2.0 凭证（桌面应用程序）
  5. 下载凭证 JSON 文件

节点类型和架构

Section titled “节点类型和架构”

Agentic Signal 使用基于节点的工作流架构，每个节点代表一个处理单元：

节点结构

Section titled “节点结构”
  • 输入/输出数据连接器 — 在节点之间连接数据流
  • 工具连接器 — AI 工具和函数的额外连接器
  • 定时器连接器 — 按时间间隔自动执行
  • 设置 — 配置节点特定选项
  • 错误日志 — 处理失败时查看错误
  • 输出 — 查看处理结果
  • 运行 — 手动触发工作流执行

可用节点类型

Section titled “可用节点类型”
节点类型描述
数据源从文件、API 或手动输入获取数据
AI 数据处理通过本地 LLM 处理数据
AI 工具使用 AI 代理执行函数调用
HTTP 数据向外部服务发起 REST API 调用
JSON 重组器转换和重构 JSON 数据
图表使用各种图表类型可视化数据
数据验证根据 schema 验证数据
数据流监控调试和检查数据流
定时器节点定期执行计划任务

日常工作流示例

Section titled “日常工作流示例”

示例 1：AI 驱动的邮件助手

Section titled “示例 1：AI 驱动的邮件助手”

创建一个工作流：

  1. 从 Gmail 获取新邮件（数据源 → Gmail）
  2. 使用本地 LLM 对邮件进行分类和优先级排序（AI 数据处理）
  3. 生成摘要回复（AI 工具）
  4. 保存到 Google Drive 供参考（HTTP 数据）

示例 2：研究自动化

Section titled “示例 2：研究自动化”

构建一个工作流：

  1. 接收研究主题作为输入（数据源）
  2. 搜索多个来源（DuckDuckGo、Brave Search 节点）
  3. 使用本地 LLM 提取和总结发现（AI 数据处理）
  4. 将结果格式化为结构化报告（JSON 重组器）
  5. 保存到文件或发送到电子邮件（输出节点）

示例 3：基于天气的通知

Section titled “示例 3：基于天气的通知”

创建每日自动化：

  1. 每天早上 7 点触发（定时器节点）
  2. 获取您所在位置的天气数据（天气 API）
  3. 使用本地 LLM 生成个性化建议（AI 数据处理）
  4. 显示格式化输出（图表或显示节点）

故障排除

Section titled “故障排除”

应用程序无法启动（Windows）

Section titled “应用程序无法启动（Windows）”

如果首次启动时应用程序缓慢或无响应：

  1. 等待杀毒软件扫描完成
  2. 关闭并重启 Agentic Signal
  3. 如有需要，向杀毒软件添加例外

Ollama 连接失败

Section titled “Ollama 连接失败”
  1. 验证 Ollama 正在运行：ollama list
  2. 检查是否启用了 CORS：OLLAMA_ORIGINS="*" ollama serve
  3. 验证应用程序设置中的 Ollama 主机为 http://localhost:11434

浏览器自动化不工作

Section titled “浏览器自动化不工作”
  1. 验证 Chrome 已安装
  2. 检查应用程序设置中的 Chrome 可执行文件路径
  3. 确保 Chrome 版本是最新的

最佳实践

Section titled “最佳实践”
  1. 从简单开始 — 在增加复杂性之前先构建基本工作流

  3. 模块化设计 — 将复杂工作流分解为更小的可重用组件
  4. 迭代测试 — 构建时运行和测试每个节点
  5. 监控性能 — 使用数据流监控调试瓶颈
