pg-aiguide：AI优化的PostgreSQL专业知识编码助手

Section titled “pg-aiguide：AI优化的PostgreSQL专业知识编码助手”

在AI驱动开发工具快速发展的今天，一个挑战始终存在：AI编码助手经常生成的数据库代码过时、缺少关键约束、不了解现代PostgreSQL功能，并且与实际最佳实践不一致。

pg-aiguide 应运而生——这是由Timescale（现Tiger Data）开发的创新MCP服务器和Claude插件，为编码助手提供AI优化的PostgreSQL专业知识。

什么是pg-aiguide？

Section titled “什么是pg-aiguide？”

pg-aiguide是一个开源工具（Apache-2.0许可），帮助AI编码工具显著改善PostgreSQL代码质量。它提供三个核心能力：

  1. 语义搜索——跨官方PostgreSQL手册（版本感知）
  2. AI优化技能——精心策划的、有主见的Postgres最佳实践，由AI代理自动使用
  3. 扩展生态系统文档，从TimescaleDB开始，更多扩展即将推出

pg-aiguide解决的问题

Section titled “pg-aiguide解决的问题”

AI编码工具经常生成的PostgreSQL代码存在以下几个问题：

  • 过时的语法——没有跟上现代PostgreSQL版本的步伐
  • 缺少约束和索引——导致数据完整性和性能问题
  • 不了解现代PG功能——错过了最近版本引入的功能
  • 与最佳实践不一致——没有遵循经验丰富的开发者的成熟模式

实际效果对比

Section titled “实际效果对比”

pg-aiguide团队进行了一项令人印象深刻的对比测试。当被要求创建一个电商数据库架构时：

  • 4倍更多的约束，相比没有pg-aiguide的AI
  • 55%更多的索引（包括部分和表达式索引）
  • PG17推荐的模式——使用最新的PostgreSQL功能
  • 现代功能如GENERATED ALWAYS AS IDENTITY和NULLS NOT DISTINCT
  • 更清晰的命名和文档——更易维护的代码

什么是 ClaudeCode Workflow Studio？

Section titled “什么是 ClaudeCode Workflow Studio？”

ClaudeCode Workflow Studio 是一个 VSCode 扩展，为 Claude Code 提供了可视化工作流编辑器。它允许你：

  • 可视化设计工作流 - 通过拖拽节点到画布上进行设计
  • 无需代码配置 - 通过 UI 设置子代理、技能和 MCP 工具
  • 一键导出 - 自动生成 .claude/agents/ 和 .claude/commands/ 文件
  • 团队协作 - 通过 Slack 分享工作流，带预览卡片

GitHub 仓库： breaking-brake/cc-wf-studio VSCode 市场： Open VSX 许可证： AGPL-3.0-or-later 版本： 3.9.1（截至 2025 年 12 月）

1. 可视化工作流编辑器

Section titled “1. 可视化工作流编辑器”

cc-wf-studio 的核心是其拖拽式画布。无需编写 markdown 和 YAML，你可以通过视觉连接节点来定义工作流逻辑：

  • MCP 节点 - 连接到模型上下文协议工具进行外部集成

Section titled “2. AI 辅助工作流优化”

  4. 重复直到满意

规划：始终从计划开始

Section titled “规划：始终从计划开始”

花五分钟规划，能节省十分钟的审查时间。

无论任务大小，都要从计划开始。无论你的代理是否有内置的”计划模式”，你都可以直接要求：

“先制定一个计划，在做出更改之前与我确认。”

规划可能会感觉像是一种开销，但它将不可避免的工作提前了。审查计划比审查代码差异要快得多。

规划决定架构形态

Section titled “规划决定架构形态”

代理的目标是以最少的编辑次数让代码工作起来。一旦某个版本存在，后续的更改只会修补已有的内容，而不是重新思考方法。计划是你有机会在代码围绕错误架构固化之前选择正确形态的机会。

异步工作流：同时处理多个任务

Section titled “异步工作流：同时处理多个任务”

与编码代理异步工作（有时称为”后台运行”）让你能够：

  • 同时处理多个任务线程
  • 避免陷入拖延
  • 让代理持续运行较长时间，而你处理其他事务

典型工作流：让代理持续运行，同时你处理其他任务，定期检查进度。

使用YOLO模式实现真正的异步

Section titled “使用YOLO模式实现真正的异步”

只有在你不批准每个工具调用的情况下，异步才能发挥作用。要么使用代理的YOLO模式，要么给它一个明确的命令允许列表，覆盖它需要的几乎所有操作。

否则你就重新发明了结对编程——而且更慢。

不同平台的命名不同：‘yolo’、‘危险地跳过权限’或类似名称。

让代码库可以进行质量验证

Section titled “让代码库可以进行质量验证”

如果代理无法测试自己的更改，它就会要求你来做——这不是对你时间的很好利用。

**投入精力让更改可以通过开发或测试命令进行验证。**并非所有内容都可以由代理测试（前端仍然需要人眼查看），但大多数更改应该以”测试通过”结束，而不是”你能试试这个吗？“

解决开发服务器问题

Section titled “解决开发服务器问题”

代理不擅长启动和清理开发服务器，这会导致：

  • 端口冲突
  • 浪费时间排查”该端口上已有内容在运行”的错误

像 Dev Manager MCP 这样的解决方案可以自动分配端口并终止僵死的服务器。

添加虚拟数据以支持离线测试

Section titled “添加虚拟数据以支持离线测试”

使用真实的种子数据让项目易于离线运行。如果代理花费一半时间让应用进入可测试状态，你将得到：

  • 不稳定的结果
  • 更慢的进度

虚拟数据还能支持并行代理——如果它们共享真实数据库，一个可能使用与另一个不同的架构版本。

不追求向后兼容

Section titled “不追求向后兼容”

每个编码代理默认都保持向后兼容。这部分是过度谨慎，部分是懒惰——保留旧接口意味着不必重构所有调用点。

将此添加到你的系统提示词中：

我们想要尽可能简单的更改。我们不关心迁移。代码可读性最重要，我们愿意做出更大的更改来实现它。

禁用禁用lint规则的功能

Section titled “禁用禁用lint规则的功能”

代理喜欢通过禁用lint规则来消除错误，而不是真正修复它们。eslint-disable-next-line 和类似模式出现在 TypeScript、Rust 以及可能的其他语言中。

在使用 Claude Code 进行软件开发时，每个会话都蕴含着宝贵的洞察——调试方法、架构决策和问题解决技巧。但这些对话被锁定在 ~/.claude/projects 目录下的 JSON 文件中，难以分享。

Simon Willison 发布了 claude-code-transcripts，这是一个 Python CLI 工具，可将这些会话文件转换为美观、可浏览的 HTML 文档，便于与他人分享。

什么是 claude-code-transcripts？

Section titled “什么是 claude-code-transcripts？”

claude-code-transcripts 是一个命令行工具，可将 Claude Code 会话数据（存储为 JSON 或 JSONL 文件）转换为分页的、移动端友好的 HTML 文档。它提供：

  • 比原始 Claude Code 界面更好的可读性
  • 可分享的输出，支持任何静态托管或 GitHub Gist
  • 多种输入源：本地会话、通过 API 导入的 Web 会话，或直接使用 JSON 文件
  • 整洁、有组织的布局，长会话自动分页

GitHub 仓库： simonw/claude-code-transcripts 作者： Simon Willison 许可证： Apache 2.0 PyPI： claude-code-transcripts
