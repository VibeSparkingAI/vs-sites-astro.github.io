Claude Code 设置

Claude Code 设置

Section titled “Claude Code 设置”

使用全局和项目级设置以及环境变量配置 Claude Code。您也可以在交互式 REPL 中通过运行 /config 命令进行配置。

设置文件

Section titled “设置文件”

settings.json 是通过分层设置配置 Claude Code 的官方机制：

  • 用户设置：~/.claude/settings.json，适用于所有项目。
  • 项目设置（在项目根目录）：
    • .claude/settings.json：可检入版本控制并与团队共享。
    • .claude/settings.local.json：本地覆盖，不应检入。创建时会自动配置 git 忽略。
  • 企业管理策略（优先于用户和项目设置）：
    • macOS：/Library/Application Support/ClaudeCode/managed-settings.json
    • Linux/WSL：/etc/claude-code/managed-settings.json
    • Windows：C:\\ProgramData\\ClaudeCode\\managed-settings.json

注意：如果同时存在多个设置文件，优先级从高到低依次为：企业管理策略 > 命令行参数 > 本地项目设置 > 共享项目设置 > 用户设置。

设置优先级

Section titled “设置优先级”

按优先级从高到低应用：

  1. 企业管理策略（managed-settings.json）——由 IT/DevOps 部署，无法覆盖。
  2. 命令行参数——特定会话临时覆盖。
  3. 本地项目设置（.claude/settings.local.json）——个人项目级偏好。
  4. 共享项目设置（.claude/settings.json）——版本控制内团队共享。
  5. 用户设置（~/.claude/settings.json）——个人全局设置。

该层次确保企业安全策略始终生效，同时允许团队和个人自定义体验。

配置系统要点

Section titled “配置系统要点”
  • 内存文件（CLAUDE.md）：启动时加载的指令与上下文。
  • 设置文件（JSON）：配置权限、环境变量与工具行为。
  • 斜杠命令：会话期间使用 /command-name 调用的自定义命令。
  • MCP 服务器：扩展 Claude Code 的外部工具与集成。
  • 优先级：高层（企业）覆盖低层（用户/项目）。
  • 继承：设置会合并，具体设置覆盖更泛设置。

系统提示可用性

Section titled “系统提示可用性”

与 claude.ai 不同，本页不发布 Claude Code 的内部系统提示。请使用 CLAUDE.md 或命令行参数 --append-system-prompt 自行扩展。

环境变量

Section titled “环境变量”

以下环境变量可用于配置 Claude Code（示例按字母序分组）：

  • ANTHROPIC_CUSTOM_HEADERS：添加自定义请求头，格式为 Name: Value。
  • ANTHROPIC_MODEL：自定义模型名称（参见模型配置）。
  • ANTHROPIC_SMALL_FAST_MODEL：后台任务使用的 Haiku 类模型名称。
  • ANTHROPIC_SMALL_FAST_MODEL_AWS_REGION：使用 Bedrock 时小型/快速模型的 AWS 区域覆盖。
  • AWS_BEARER_TOKEN_BEDROCK：Bedrock API 密钥（参见 Bedrock API 密钥）。
  • BASH_DEFAULT_TIMEOUT_MS：长时运行 bash 命令的默认超时。
  • BASH_MAX_TIMEOUT_MS：模型可为长时运行 bash 命令设置的最大超时。
  • BASH_MAX_OUTPUT_LENGTH：bash 输出在中间截断前的最大字符数。
  • CLAUDE_BASH_MAINTAIN_PROJECT_WORKING_DIR：每个 bash 命令后返回原始工作目录。
  • CLAUDE_CODE_API_KEY_HELPER_TTL_MS：使用 apiKeyHelper 时凭据刷新的间隔（毫秒）。
  • CLAUDE_CODE_IDE_SKIP_AUTO_INSTALL：跳过 IDE 扩展的自动安装。
  • CLAUDE_CODE_MAX_OUTPUT_TOKENS：大多数请求的最大输出 tokens。
  • CLAUDE_CODE_USE_BEDROCK：启用 Bedrock。
  • CLAUDE_CODE_USE_VERTEX：启用 Vertex。
  • CLAUDE_CODE_SKIP_BEDROCK_AUTH：跳过 Bedrock 的 AWS 身份验证（如经由 LLM 网关）。
  • CLAUDE_CODE_SKIP_VERTEX_AUTH：跳过 Vertex 的 Google 身份验证（如经由 LLM 网关）。
  • CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC：等同设置 DISABLE_AUTOUPDATER、DISABLE_BUG_COMMAND、DISABLE_ERROR_REPORTING、DISABLE_TELEMETRY。
  • CLAUDE_CODE_DISABLE_TERMINAL_TITLE：设为 1 禁用基于上下文的自动终端标题更新。
  • DISABLE_AUTOUPDATER：设为 1 禁用自动更新（优先于 autoUpdates 配置）。
  • DISABLE_BUG_COMMAND：设为 1 禁用 /bug 命令。
  • DISABLE_COST_WARNINGS：设为 1 禁用成本警告。
  • DISABLE_ERROR_REPORTING：设为 1 退出 Sentry 错误报告。
  • DISABLE_NON_ESSENTIAL_MODEL_CALLS：设为 1 禁用非关键模型调用（如风味文本）。
  • DISABLE_TELEMETRY：设为 1 退出 Statsig 遥测（不含代码/路径/命令数据）。
  • HTTP_PROXY：HTTP 代理地址。
  • HTTPS_PROXY：HTTPS 代理地址。
  • MAX_THINKING_TOKENS：限制模型的思考预算。
  • MCP_TIMEOUT：MCP 服务器启动超时（毫秒）。
  • MCP_TOOL_TIMEOUT：MCP 工具执行超时（毫秒）。
  • MAX_MCP_OUTPUT_TOKENS：MCP 工具响应允许的最大 tokens（默认 25000，超过 10000 显示警告）。
  • USE_BUILTIN_RIPGREP：设为 0 使用系统 rg 而非内置 rg。
  • VERTEX_REGION_CLAUDE_3_5_HAIKU：Vertex AI 下 Claude 3.5 Haiku 区域覆盖。
  • VERTEX_REGION_CLAUDE_3_5_SONNET：Vertex AI 下 Claude 3.5 Sonnet 区域覆盖。
  • VERTEX_REGION_CLAUDE_3_7_SONNET：Vertex AI 下 Claude 3.7 Sonnet 区域覆盖。
  • VERTEX_REGION_CLAUDE_4_0_OPUS：Vertex AI 下 Claude 4.0 Opus 区域覆盖。
  • VERTEX_REGION_CLAUDE_4_0_SONNET：Vertex AI 下 Claude 4.0 Sonnet 区域覆盖。
  • VERTEX_REGION_CLAUDE_4_1_OPUS：Vertex AI 下 Claude 4.1 Opus 区域覆盖。
